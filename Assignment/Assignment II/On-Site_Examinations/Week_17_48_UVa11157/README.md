# UVa 11157 - Dynamic Frog

### 1. 題目摘要
有一隻青蛙站在編號 1 的石頭上，平面上共有 $N$ 顆石頭，每顆石頭有自己的座標。青蛙必須從第 1 顆石頭出發，依序跳過「每一顆石頭恰好一次」（即走訪一條 Hamiltonian Path）。每次跳躍的「代價」定義為兩石頭間的歐式距離平方。
請找出一種拜訪順序，使得「過程中所有跳躍代價的最大值（瓶頸值）」最小化；若有多種順序同樣能達到此最小瓶頸值，輸出「字典序最小」的那一條路徑。

### 2. 我的解題邏輯
這是經典的「最小化最大邊權的 Hamiltonian Path（Bottleneck Hamiltonian Path）」問題，由於 $N$ 很小（題目限制 $N \le 10$），可以用「狀態壓縮 DP + 二分搜尋門檻值」來解決。

1. **建立距離矩陣**：
   計算任兩石頭間的歐式距離平方 `cost[i][j]`，避免使用浮點數造成誤差。

2. **窮舉門檻值 (Threshold)**：
   把所有 $N^2$ 個距離值收集起來、去重排序，依序由小到大嘗試每個候選值 $T$，檢查「只使用代價 $\le T$ 的邊，是否能從石頭 1 出發走訪所有石頭」。第一個讓檢查成立的 $T$，就是答案的最小瓶頸值 `bestT`。

3. **可行性檢查 (Bitmask DP)**：
   定義 `feas[mask][last]` 表示「目前已走訪過 `mask` 集合的石頭、目前停在 `last`，是否能在只使用代價 $\le T$ 的邊的前提下，繼續走完剩下所有石頭」。
   * 邊界：`mask == full`（全部走完）時為真。
   * 轉移：嘗試所有尚未走訪、且 `cost[last][v] <= T` 的下一顆石頭 `v`，只要有一種選擇能讓 `feas[mask|bit(v)][v]` 為真，則 `feas[mask][last]` 為真。
   * 用記憶化遞迴（memoization）實作，避免重複計算。

4. **重建字典序最小路徑**：
   找到 `bestT` 後，從起點（mask=只有石頭1, last=石頭1）開始，每一步「由小到大」嘗試下一個石頭編號 `v`：
   * 若 `cost[last][v] <= bestT` 且 `feas[mask|bit(v)][v]` 為真，就代表選擇 `v` 之後仍能完成整趟旅程，於是貪婪地選擇這個 `v`（因為是由小到大嘗試，第一個符合條件的就是字典序最小的選擇）。
   * 重複此步驟直到走訪完所有石頭，即可得到字典序最小的最佳路徑。

### 3. 筆記
* **為什麼要先二分/線性搜尋門檻值，而非直接 DP 求最小瓶頸**：
  瓶頸路徑問題（最小化最大邊權）的特性是：一旦固定一個門檻 $T$，問題就退化成「是否存在一條只用代價 $\le T$ 的邊構成的 Hamiltonian Path」的可行性判定問題，這比直接在 DP 過程中同時追蹤「目前路徑最大值」更簡單、狀態數也更少（不需要把「目前最大代價」也塞進狀態），因此採用「窮舉門檻值 + 可行性 DP」的組合策略。

* **為什麼可行性檢查要用記憶化而非單純 BFS/DFS**：
  因為同一個 `(mask, last)` 狀態在路徑重建與門檻檢查過程中會被重複查詢非常多次，用 `feas[mask][last]` 記憶化可以保證每個狀態只計算一次，避免指數級的重複計算。

* **時間複雜度**：
  狀態總數為 $O(2^N \times N)$，每個狀態轉移最多檢查 $N$ 個候選下一步，因此單次可行性檢查為 $O(2^N \times N^2)$。外層需要對最多 $O(N^2)$ 個候選門檻值做檢查，整體最壞時間複雜度約為 $O(N^2 \times 2^N \times N^2) = O(N^4 \times 2^N)$，在 $N \le 10$ 的限制下絕對可以快速跑完。

* **字典序最小的實作技巧**：
  在重建路徑時，**永遠由小到大檢查候選的下一顆石頭**，第一個「代價合法且可行」的選擇即為當前最優，這保證了整條路徑在所有等價最優解中是字典序最小的，不需要額外回溯或比較多條路徑。

* **多筆測試資料 / 結束條件**：
  當讀入 $N = 0$ 時代表輸入結束，程式立即停止。每組測試案例之間視題目要求以空行分隔輸出。
