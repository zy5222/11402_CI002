# UVa 10020 - Minimal coverage

### 1. 題目摘要
給定一條目標線段 $[0, L]$，以及 $N$ 條候選線段（用整數座標 $[x, y]$ 表示），輸入以 `-1 -1` 結束該測試案例的線段列表。請選出**最少數量**的候選線段，使其聯集能完整覆蓋 $[0, L]$。
若可以覆蓋，輸出所選線段數量與每條線段（依選用順序）；若無法覆蓋，輸出 `0`。

### 2. 我的解題邏輯
這是經典的「區間覆蓋」貪婪策略題：每一步都選擇「能讓覆蓋範圍延伸最遠」的線段。

1. **排序 (Sorting)**：
   將所有候選線段依左端點 $x$ 由小到大排序。

2. **貪婪延伸 (Greedy Extension)**：
   維護一個變數 `covered` 表示「目前已確定覆蓋到的最右位置」，初始為 $0$。
   重複以下步驟直到 `covered >= L`：
   * 利用指標掃描所有「左端點 $\le$ covered」的候選線段（因為這些線段才能銜接上目前已覆蓋的範圍）。
   * 在這些候選線段中，挑出右端點 $y$ 最大的那一條，作為這一步的最佳選擇。
   * 若找不到任何可銜接的線段（候選線段左端點都大於 `covered`），代表出現「斷層」，無法覆蓋，直接判定失敗。
   * 否則將該線段加入答案，並把 `covered` 更新為它的右端點。

3. **終止判定**：
   * 若 `covered` 成功達到或超過 $L$，輸出所選線段數量與內容。
   * 若中途出現無法銜接的斷層，輸出 `0`。

### 3. 筆記
* **為什麼貪婪會成功？**：
  在所有「左端點 $\le$ covered」的合法候選中，選右端點最大的線段必然不劣於選其他線段——它能讓下一步的搜尋範圍最大，後面能選的線段只會更多或相同，不會更少。這是典型的「交換論證 (exchange argument)」可證明貪婪最優。

* **指標不回退的正確性**：
  雖然候選線段是用單一指標 `i` 持續往前掃描而不重置，但因為線段已依左端點排序，且 `covered` 只會單調遞增，所以每條線段只會被檢查一次、不會被重複或遺漏考慮。

* **時間複雜度**：
  排序耗時 $O(N \log N)$，後續貪婪掃描每條線段最多處理一次，耗時 $O(N)$，總時間複雜度為 $O(N \log N)$。

* **輸出格式細節**：
  各測試案例之間需要以空行分隔（但第一個案例前不能有多餘空行），這點容易因格式問題（Presentation Error）而被扣分，需特別注意。
