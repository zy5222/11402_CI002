# UVa 12365 - Army Buddies

### 1. 題目摘要

戰場上有 $S$ 名士兵排成一列，編號為 1 到 $S$。當發生戰鬥時，從編號 $L$ 到 $R$ 的區間內的士兵會陣亡。陣亡後，倖存的左右鄰居會重新連繫起來。給定 $B$ 次戰鬥事件，每次事件給出陣亡區間 $[L, R]$，請輸出每次事件後，左邊倖存者的編號（若無則輸出 `*`）以及右邊倖存者的編號（若無則輸出 `*`）。

### 2. 解題邏輯

這題若使用陣列紀錄每個人的存活狀態並線性搜尋，時間複雜度會高達 $O(S \times B)$，導致 Time Limit Exceeded。我們需要以 $O(1)$ 的速度找到新的左右鄰居。

1. **連結串列模擬**：
   我們使用兩個陣列 `L[S+1]` 和 `R[S+1]` 分別記錄每個編號士兵的左右鄰居。
2. **快速查詢**：
   * `L[i]` 初始為 `i-1`，`R[i]` 初始為 `i+1`。
   * 當 $[L, R]$ 區間陣亡時，新的左鄰居是 `L[L]`，新的右鄰居是 `R[R]`。
3. **更新鄰居關係**：
   更新邏輯為：
   * `R[L[L]] = R[R]`
   * `L[R[R]] = L[L]`
   這兩行程式碼直接將 $[L, R]$ 這個斷裂區間的左右兩端「縫合」。若縫合處超出邊界（即 0 或 $S+1$），則顯示 `*`。

### 3. 筆記

* **時間複雜度**：每次戰鬥事件僅需常數時間進行指標更新，總時間複雜度為 $O(B)$。即便 $S$ 高達 10 萬，也能在瞬間完成。
* **空間複雜度**：只需要兩個長度為 $S$ 的陣列，空間複雜度為 $O(S)$。
