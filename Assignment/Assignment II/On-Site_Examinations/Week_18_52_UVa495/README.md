# UVa 495 - Fibonacci Freeze

### 1. 題目摘要
請計算第 $N$ 個費氏數 (Fibonacci number) 的值，其中 $0 \le N \le 5000$。由於 $N$ 範圍很大，答案會是極為龐大的數字，遠超出一般整數型別所能容納的範圍。輸出格式為：`The Fibonacci number for N is F(N)`。

### 2. 我的解題邏輯
這題與 UVa 10359 (Tiling) 解法幾乎完全相同，都是「標準 Fibonacci 遞迴 + 大數加法」的組合，差別只在於 $N$ 的範圍更大（最大到 5000），以及輸出格式不同。

1. **標準 Fibonacci 遞迴關係**：
   定義 $f(0) = 0$、$f(1) = 1$，遞迴關係為：
   $$f(N) = f(N-1) + f(N-2)$$
   這是最經典的費氏數列定義方式。

2. **大數加法 (Big Integer Addition)**：
   由於 $N$ 最大到 5000，$f(5000)$ 的位數會非常龐大（約 1000 多位數），遠遠超出任何固定大小整數型別所能表示的範圍，因此必須採用「陣列模擬大數」的方式儲存與相加。
   * 用 `vector<int>` 儲存大數，每個元素代表一個十進位數字，索引 0 為個位數，方便逐位相加與處理進位。
   * `add` 函式模擬手動直式加法，逐位相加並處理進位（carry）。

3. **預先計算所有答案**：
   由於測試資料可能有多筆查詢，且 $N$ 範圍固定（$0 \le N \le 5000$），採用「預先計算 $f(0)$ 到 $f(5000)$ 所有結果並存入陣列」的策略，每筆查詢只需 $O(1)$ 直接查表並輸出，避免重複計算造成效能浪費。

4. **輸出格式處理**：
   依照題目要求的固定句型 `The Fibonacci number for N is F(N)` 來輸出，將查表得到的大數結果透過 `toString` 函式轉換為字串後接在句尾輸出。

### 3. 筆記
* **與 UVa 10359 的差異**：
  解題邏輯與遞迴式完全相同（都是標準 Fibonacci 定義），主要差異在於：
  * 資料範圍從 $N \le 250$ 大幅擴大到 $N \le 5000$，使得答案位數從約 50~60 位暴增到超過 1000 位，大數陣列與加法運算量也隨之大幅增加。
  * 輸出格式不同，這題需要包出完整句子，而非單純輸出數字本身，需特別注意句型與標點符號是否正確（包括大小寫與單複數）。

* **大數運算的效能考量**：
  雖然單次大數加法的時間複雜度與位數成正比（最壞約 1000+ 位數），但因為只需要在程式開始時「預先計算一次」（共 5000 次加法），總計算量約為 $O(5000 \times 1000) = O(5 \times 10^6)$ 量級的基本運算，效能上完全沒有問題，遠快於對每筆測資都重新計算一次。

* **陣列大小設置的注意事項**：
  由於需要查到 $f(5000)$，宣告大數陣列 `f` 時務必確保大小至少為 `5001`（索引 0 到 5000 都要能存取），避免發生陣列越界（index out of range）的執行期錯誤。

* **多筆測試資料 / 結束條件**：
  此題沒有明確的結束標記，直接讀到檔案結尾 (EOF) 為止，因此使用 `while(cin >> N)` 持續讀取直到讀取失敗（EOF）才結束迴圈。
