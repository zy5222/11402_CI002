# UVa 1062 - Containers

### 1. 題目摘要

港口有一批貨櫃依序抵達，貨櫃上標有 `A` 到 `Z` 的英文字母。你需要將這些貨櫃堆疊起來，但有一個嚴格的規定：**只有當新進貨櫃的字母「小於或等於」堆疊最上方的字母時，才能疊上去**（例如 `B` 可以疊在 `C` 或 `B` 上面，但不能疊在 `A` 上面）。
如果找不到任何符合條件的堆疊，就必須為這個新貨櫃建立一個新的堆疊。
請求出最少需要幾個堆疊才能安置所有的貨櫃。當輸入為 `end` 時程式結束。

### 2. 解題邏輯

這題的核心概念是**「貪婪演算法 (Greedy)」**，其運作方式剛好等同於著名的**「耐心排序 (Patience Sorting)」**。為了讓堆疊數量最少，我們應該盡可能把貨櫃疊在現有的堆疊上。

1. 我們建立一個陣列 `tops`，用來記錄「目前每個堆疊最上方的貨櫃字母」。
2. 依序讀取字串中的每一個字母（代表新進貨櫃）。
3. **尋找合適的堆疊**：從左到右掃描 `tops` 陣列，尋找**第一個**字母大於或等於當前貨櫃字母的堆疊。
   * 如果找到了：就把這個貨櫃疊上去（更新該堆疊的最上方字母為當前字母）。
   * 如果找不到：代表現有堆疊都無法容納這個貨櫃，我們只好擴建一個新的堆疊（將當前字母加入 `tops` 的尾端）。
4. 最後，`tops` 陣列的長度，就是我們最少需要的堆疊總數。

### 3. 筆記

* **為什麼找「第一個」就夠了？**：
  因為我們是依序建立堆疊的，`tops` 陣列裡的字母在邏輯上會自動維持一種「非嚴格遞增」的趨勢。把貨櫃貪婪地放在最左邊能放的地方，就能保留右邊較大字母的彈性，這是耐心排序的最佳策略。
* **與 LIS 的關聯**：
  這題求的「最少堆疊數」，在數學本質上其實等價於原字串的**「最長嚴格遞增子序列 (Longest Strictly Increasing Subsequence)」**的長度。
* **時間複雜度**：
  最多只有 26 種英文字母，所以堆疊數量最多不會超過 26 個。對於長度為 N 的字串，每次尋找只需掃描最多 26 個元素，時間複雜度為 O(N)，執行速度極快。
