# UVa 900 - Brick Wall Patterns

### 1. 題目摘要
有一道高度為 2、寬度為 $N$ 的牆，要用大小為 $1 \times 2$ 的磚塊（可橫放也可直放）完全鋪滿。請計算總共有多少種不同的鋪法，其中 $N$ 最大可達 30，答案可能會是非常大的數字（超出一般整數型別範圍）。

### 2. 我的解題邏輯
這是經典的「骨牌鋪磚」遞迴關係問題，核心是推導出 $f(N)$（寬度為 $N$ 時的鋪法數）與較小子問題之間的遞迴關係。

1. **遞迴關係推導**：
   考慮寬度為 $N$ 的牆，觀察最右側（第 $N$ 欄）的鋪法情況：
   * **情況一**：用一塊磚直放在最右側那一欄（佔滿該欄的上下兩格），剩下寬度 $N-1$ 的牆需要鋪滿，貢獻 $f(N-1)$ 種方法。
   * **情況二**：用兩塊磚橫放，分別佔據最右側兩欄的上排與下排，剩下寬度 $N-2$ 的牆需要鋪滿，貢獻 $f(N-2)$ 種方法。
   
   因此可得遞迴式：
   $$f(N) = f(N-1) + f(N-2)$$
   
   邊界條件為 $f(1) = 1$（只能直放一塊磚）、$f(2) = 2$（可以直放兩塊，或橫放兩塊）。
   
   這個數列本質上就是「Fibonacci 數列」的變形（偏移版本）。

2. **大數加法 (Big Integer Addition)**：
   由於 $N$ 最大到 30，雖然遞增速度不像階乘那麼快，但仍可能超出一般整數範圍（視具體數值規模而定），因此採用「字串/陣列模擬大數」的方式來儲存與相加 $f(N)$，確保不會發生溢位問題。
   * 用 `vector<int>` 儲存大數，每個元素代表一個十進位數字，索引 0 為個位數（方便進位運算）。
   * 實作 `add` 函式模擬手動加法的逐位相加、進位機制。

3. **預先計算所有答案**：
   由於 $N \le 30$ 且測試資料可能有多筆，採用「預先計算 $f(1)$ 到 $f(30)$ 所有結果並存入陣列」的策略，這樣每筆查詢只需要 $O(1)$ 直接查表並輸出，不需要每次重新計算。

### 3. 筆記
* **為什麼答案是 Fibonacci 數列的變形**：
  雖然遞迴式 $f(N) = f(N-1) + f(N-2)$ 與標準 Fibonacci 數列形式相同，但由於邊界條件不同（$f(1)=1, f(2)=2$，而非標準 Fibonacci 的 $1, 1$），所以實際上 $f(N)$ 對應到的是標準 Fibonacci 數列中往後平移一位的數值（即 $f(N) = \text{Fib}(N+1)$，其中 Fib 為標準定義）。

* **大數運算的必要性**：
  雖然題目 $N$ 不算特別大，但題目本身明確提示答案可能超出標準整數型別容納範圍，且大數加法的實作成本很低（只是模擬國小學過的直式加法），因此直接採用大數運算可以一勞永逸地避免任何潛在的溢位風險，不需要額外判斷何時該切換成大數模式。

* **時間複雜度**：
  預先計算 30 個大數加法，每次加法操作的時間複雜度與數字位數成正比（位數最多約十位數），整體預處理時間可忽略不計；每筆查詢輸出為 $O(1)$ 查表，因此即使有大量測試資料，程式依然能非常快速地完成。

* **多筆測試資料 / 結束條件**：
  此題沒有明確的結束標記，直接讀到檔案結尾 (EOF) 為止，因此使用 `while(cin >> N)` 持續讀取直到讀取失敗才結束迴圈。
