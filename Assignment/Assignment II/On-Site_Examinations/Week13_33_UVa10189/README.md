# UVa 10189 - Minesweeper

### 1. 題目摘要

這題就是經典的「踩地雷」遊戲。
給定一個 $N \times M$ 的網格，其中 `*` 代表地雷，`.` 代表安全的空地。
請你將所有的空地替換成一個數字（0 到 8），這個數字代表該空地周圍的 8 個方向（上下左右與四個斜角）總共藏了幾顆地雷。當輸入為 `0 0` 時結束程式。

### 2. 解題邏輯

這題不需要複雜的演算法，考驗的是純粹的「二維陣列走訪」與「邊界判斷」實作能力。

1. **讀取網格**：將每組測資的 $N$ 行字串存入 `vector<string>` 當中。
2. **方向陣列 (Direction Array)**：為了避免寫 8 個冗長的 `if` 來判斷周圍的格子，我們可以建立兩個陣列 `dx` 和 `dy`，用來代表 8 個方向的座標偏移量。
3. **走訪與計算**：
   * 使用雙層迴圈掃描整個網格。
   * 如果當前格子是 `*`，直接印出 `*`。
   * 如果當前格子是 `.`，則跑一個跑 8 次的迴圈，搭配方向陣列去檢查周圍的格子。
4. **邊界防護**：在檢查周圍格子時，必須先確認計算後的新座標是否超出網格範圍（`ni >= 0 && ni < n && nj >= 0 && nj < m`），才去讀取裡面的字元，否則會發生記憶體越界錯誤 (Segfault)。

### 3. 筆記

* **最惡名昭彰的 PE 陷阱**：
  這題的輸出格式有一個極度容易讓人吃 Presentation Error (PE) 的陷阱。題目要求「每兩組測資之間必須有一個空白行」，這句話的弦外之音是：**最後一組測資的結尾不能有多餘的空白行**。實作時，我們可以使用一個 `bool first` 變數來控制，只有在「不是第一組測資」準備印出時，才先印出一個換行符號。
* **時間與空間複雜度**：
  每個格子只會被周圍 8 個方向檢查一次，時間複雜度為 $O(N \times M)$。由於 $N$ 和 $M$ 最大只有 100，運算次數極少，效能非常高。
* **字串取代字元**：
  利用 C++ 的 `vector<string>` 來儲存輸入，比起宣告傳統的 `char grid[100][100]` 更不容易出錯，讀取也更簡潔。
