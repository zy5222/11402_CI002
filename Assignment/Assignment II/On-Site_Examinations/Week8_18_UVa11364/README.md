# UVa 11364 - Parking

### 1. 題目摘要

主角要在一直線的街道上停車，然後走路去逛完清單上的每一家店，最後再走回車上。
我們需要算出：不管他把車停在哪個位置，他所需要走的「**最短總步行距離**」是多少？

### 2. 我的解題邏輯

這題的關鍵是**車停在哪裡根本不重要**（只要停在最兩端的店鋪之間就好）。
因為不管從哪裡出發，你絕對必須走到「最左邊的店」，也一定得走到「最右邊的店」，最後還要折返回車上。也就是說，最左邊到最右邊的這段距離，一定會被完整地走過「一趟去、一趟回」。

所以我的做法很直覺：
1. **儲存座標**：開一個大小為 `n` 的 `vector`，把所有店的座標都讀進來。
2. **排序找兩端**：直接套用 `<algorithm>` 裡的 `sort(v.begin(), v.end())` 將座標從小到大排好。
3. **套用公式**：排好序之後，陣列的第一個數字 `v[0]` 就是最左邊的店，最後一個數字 `v[n - 1]` 就是最右邊的店。直接計算 `(最右 - 最左) * 2` 就是最終答案。

### 3. 筆記與心得

* **直覺的 Sort 找極值法**：
  這題其實可以不用陣列，邊讀邊用 `max()` 和 `min()` 來找。但我這次選擇直接存進 `vector` 裡面 `sort`。
* **小心陷阱題**：
  一開始看到題目可能會想去算什麼平均值或中位數來找「最佳停車位」，但其實完全不需要。這題本質就是求 `區間長度 * 2` 是最大的收穫。
