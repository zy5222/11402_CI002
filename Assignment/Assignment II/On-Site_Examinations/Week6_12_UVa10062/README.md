# UVa 10062 - Tell me the frequencies!

## 1. 題目摘要
給定好幾行字串，要求統計裡面每一個 ASCII 字元出現的次數。
最後要把結果印出來，排序的規則是：
1. 出現次數少的排前面（由小到大）。
2. 如果出現次數一樣多，則 ASCII 碼「較大」的排前面（由大到小）。

## 2. 解題邏輯
這題跟 UVa 10008 概念幾乎一樣，所以我繼續沿用「雙層迴圈掃描法」來避開寫 `struct` 跟自定義排序的麻煩。

* **頻率統計**：
  直接開一個大小為 256 的陣列 `freq`（涵蓋所有 ASCII 碼），把讀到的字元當成陣列的 Index 直接 +1 (`freq[line[i]]++`)。
* **核心方法 (取代 sort)**：
  - **外層迴圈** 控制頻率 `f`：從出現 `1` 次開始，慢慢往上找到字串的最大長度。（這滿足了條件 1：頻率由小到大）。
  - **內層迴圈** 控制字元 `i`：從 ASCII 碼 `255` 開始，一路往下扣到 `0`。（這滿足了條件 2：當頻率 `f` 對撞相同時，ASCII 大的會先被找到並印出來）。
  - 只要 `freq[i] == f`，就代表找到了，直接印出ASCII 數值和頻率。

## 3. 注意事項與筆記
- **注意Presentation Error**：題目要求「兩筆測資的輸出結果之間要有一個空行」，但是「最後一筆測資結尾不能有多的空行」。我用了一個 `bool first = true` 的開關，如果是第一筆就不印換行，第二筆開始才在「最前面」印換行，可以解決這個問題。
- **空白也是字元**：因為字串裡面可能會包含空白鍵，所以絕對不能用 `cin >>`，一定要乖乖用 `getline(cin, line)` 才能正確統計。
