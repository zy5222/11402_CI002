# UVa 11063 - B2-Sequence

### 1. 題目摘要

給一個長度為 $n$ 的數列，判斷它是不是「B2-Sequence」。
要滿足這個條件有三個重點：
1. **正整數**：陣列裡的每一個數字都必須 $\ge 1$。
2. **嚴格遞增**：後面的數字一定要比前面的大。
3. **兩兩總和唯一**：從數列裡隨便挑兩個數字相加（包含自己加自己），算出來的總和絕對不能重複。

### 2. 我的解題邏輯

這題要檢查「有沒有重複出現的總和」，我用`set` 來做，這樣就不用自己開大陣列慢慢搜，省下很多麻煩。

* **邊讀邊防呆**：
  在 `cin` 讀進陣列的時候，我先檢查了 `b[i] < 1`。只要出現 0 或負數，就馬上把 `isB2` 標記成 `false`。
* **檢查遞增趨勢**：
  跑一個簡單的單層迴圈檢查 `b[i] >= b[i + 1]`，只要發現沒有嚴格遞增，一樣判 `false`。
* **核心處理（檢查總和）**：
  用雙層 `for` 迴圈把所有組合加起來。要注意內層迴圈的 `j` 必須從 `i` 開始，這樣才能涵蓋到「同一個 index 加自己」的狀況。
  每次算出 `num = b[i] + b[j]` 後，我用 `sums.count(num)` 去看 `set` 裡面是不是已經有這個數字了。如果有，代表加總重複，`isB2 = false`；如果沒有，就用 `sums.insert(num)` 把它加進集合裡繼續測。

### 3. 筆記與待改進的地方

* **陷阱題定義看清楚**：我發現這題最大的坑在於「忘記檢查正整數」。題目第一句定義就寫了 $1 \le b_1 < b_2 \dots$，如果測資故意塞 `0` 進來，沒判斷到就會錯。
* **關於效能的檢討**：前面檢查發現不是正整數或是沒遞增（`isB2` 已經是 `false` 了），下面那坨雙層迴圈還是會照樣硬跑完。如果之後測資很大，容易 TLE。下次重寫應該要在進入雙層迴圈前包一個 `if (isB2)`，而且內層發現重複時應該要直接 `break`。
