# UVa 299 - Train Swapping

## 1. 題目摘要
有一列火車的車廂順序被打亂了，車廂編號是 1 到 L。現在我們要把車廂排回由小到大的正確順序，但唯一的動作只能「把相鄰的兩個車廂對調」。
題目會給定車廂的初始排列狀態，要求我們算出最少需要對調幾次，才能把整列火車排好。

## 2. 解題邏輯
這題核心是利用**氣泡排序法 (Bubble Sort)** (不斷交換相鄰的元素)。

* **資料儲存**：一開始先讀入總測資數 `n`，然後每一筆測資先讀取車廂長度 `l`，並宣告一個大小為 `l` 的 `vector` 來裝車廂號碼。
* **氣泡排序與計數**：
  * 宣告一個計數器 `t = 0`。
  * 寫兩個標準的 `for` 迴圈跑氣泡排序。內層迴圈如果發現前面的數字大於後面的數字 (`train[j] > train[j + 1]`)，就執行 `swap` 交換它們。
  * **關鍵點**：每次執行 `swap` 的同時，就把計數器 `t` 加 1。
* 當雙層迴圈跑完，陣列不僅排好了，`t` 裡面存的數字剛好就是我們總共交換的次數，直接印出來收工！

## 3. 注意事項與筆記
- **逆序對 (Inversions)**：這題在演算法的專有名詞叫做求「逆序對」。雖然用 Merge Sort 可以做到更快  $O(N \log N)$，但這題的車廂長度 $L$ 最大才 50，直接用 $O(N^2)$ 的氣泡排序法。
