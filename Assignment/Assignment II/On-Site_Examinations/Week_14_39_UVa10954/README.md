# UVa 10954 - Add All

### 1. 題目摘要

給定 $N$ 個正整數，你需要將它們全部加總起來。
每次相加「兩個數字」時，會產生等同於「這兩個數字之和」的「花費 (Cost)」。相加後產生的新數字會放回數字堆中，繼續與其他數字相加，直到只剩下一個數字為止。
請找出將所有數字加總的「**最小總花費**」。當輸入為 0 時結束程式。

### 2. 解題邏輯

這是一題標準的**貪婪演算法 (Greedy)**。
因為每一次相加產生的「和」，都會在後續的步驟中被反覆加進總花費裡，所以我們必須讓**越大的數字越晚被加到，越小的數字越早被加到**。

最佳策略：**每次都挑選當前數字堆中「最小的兩個數字」來相加。**

1. 建立一個**最小堆積 (Min-Heap)**，在 C++ 中可以使用 `priority_queue` 配合 `greater` 來實作。
2. 將所有數字推入這個 Min-Heap 中。
3. 只要 Heap 裡面的數字數量大於 1，就執行以下操作：
   * 取出並移除 Heap 中最小的兩個數字。
   * 將這兩個數字相加，得到一個新的 `sum`。
   * 把這個 `sum` 累加到全域的 `total_cost` 中。
   * 把這個 `sum` 重新推回 Heap 裡面，參與後續的相加。
4. 當 Heap 只剩下一個數字時，`total_cost` 就是我們的最小總花費。

### 3. 筆記

* **資料型態防禦 (Overflow)**：
  雖然題目給定的單一數字最大為 100,000，且 $N \le 5000$，但在不斷相加的過程中，最後的 `total_cost` 或是中間產生的 `sum` 是有可能超過 32-bit `int` 的極限的。因此，計算花費的變數與優先佇列的型態，強烈建議直接開 `long long` 來避免整數溢位 (Integer Overflow)。
* **時間與空間複雜度**：
  每次從優先佇列取出與推入元素的時間複雜度為 $O(\log N)$。總共需要執行約 $N$ 次合併操作，所以整體時間複雜度為 $O(N \log N)$。空間複雜度為 $O(N)$（用來儲存一開始的 $N$ 個數字）。
