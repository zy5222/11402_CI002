# UVa 102 - Ecological Bin Packing

### 1. 題目摘要
有三個垃圾桶（編號 0、1、2），每個垃圾桶裡裝有三種瓶子：棕色 (Brown)、綠色 (Green)、透明 (Clear) 的數量各不相同。
規則要求每個垃圾桶最終只能裝「同一種顏色」的瓶子。請決定一種分配方式（每個垃圾桶指定一種要保留的顏色，且三個垃圾桶分別對應到三種不同顏色），使得「需要搬移出垃圾桶外的瓶子總數」最少。
輸出格式為：三個字母代表三個垃圾桶（依序）所保留的瓶子顏色（B/C/G），以及最少搬移數量。

### 2. 我的解題邏輯
這題的關鍵觀察是：**只有 3 個垃圾桶與 3 種顏色，所以對應方式只有 $3! = 6$ 種排列**，可以直接暴力枚舉所有排列找最佳解，不需要複雜演算法。

1. **輸入資料整理**：
   讀入一個 $3 \times 3$ 的矩陣 `cnt[bin][type]`，其中 `bin` 為垃圾桶編號（0, 1, 2），`type` 為瓶子種類（0=Brown, 1=Clear, 2=Green，依照輸入順序）。
   同時計算每個垃圾桶的總瓶數 `total[bin]`。

2. **枚舉所有排列 (Permutation)**：
   使用 `next_permutation` 枚舉 `perm[3]` 陣列的所有排列，其中 `perm[bin]` 代表「第 `bin` 個垃圾桶要保留的瓶子種類」（這是一個 0~2 的全排列，確保三個垃圾桶對應到三種不同顏色，不會重複）。

3. **計算搬移成本**：
   對於某一種排列方案，第 `bin` 個垃圾桶要保留 `perm[bin]` 這個種類，因此「需要搬出去」的瓶子數為：
   $$\text{cost}_{bin} = \text{total}[bin] - \text{cnt}[bin][\text{perm}[bin]]$$
   將三個垃圾桶的搬移成本加總，即為這個排列方案的總成本。

4. **找出最佳排列**：
   在 6 種排列中，找出總成本最小的那一種，記錄下它的 `bestPerm` 陣列與 `bestCost`。

5. **組合輸出字串**：
   依照 `bestPerm[bin]` 所代表的種類（0, 1, 2 對應到 B, C, G），組合出三個字母的字串（例如 `BCG`），代表三個垃圾桶分別保留的顏色，並輸出該字串與最小成本 `bestCost`。

### 3. 筆記
* **為什麼可以暴力枚舉**：
  因為垃圾桶數量與顏色種類都固定為 3，排列數量僅有 $3! = 6$ 種，對任何資料規模而言都是常數時間，完全不需要設計動態規劃或其他最佳化演算法，直接窮舉即是最簡單且最穩健的解法。

* **搬移成本的意義**：
  「保留某種顏色」代表「不需要搬出去的部分」，所以該垃圾桶的搬移成本 = 該垃圾桶總瓶數 - 保留顏色的瓶數。這是貪婪概念以外的「枚舉 + 局部計算成本」典型應用。

* **輸入順序與顏色對應**：
  題目原始輸入順序為 Brown、Clear、Green（與許多人直覺的 B-G-C 不同），需特別注意 `names` 陣列裡字母與索引的對應關係（`names[0]="BCG"` 代表索引 0,1,2 分別對應 B, C, G），避免輸出時顏色字母對應錯誤。
