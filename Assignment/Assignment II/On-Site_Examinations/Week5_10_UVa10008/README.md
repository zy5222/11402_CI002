# UVa 10008 - What's Cryptanalysis?

## 1. 題目摘要
給定 $n$ 行字串，要求統計裡面所有英文字母出現的次數（大小寫視為相同，都要算在同一組）。
最後要把統計結果印出來，排序規則很嚴格：
1. 出現次數多的排前面（由大到小）。
2. 如果出現次數一樣多，則按照字母順序排（由 A 到 Z）。

## 2. 解題邏輯
這題可以拆成「統計」跟「排序輸出」兩個部分來看：

* **統計字母 (`code` 陣列)**：
  - 開一個大小為 26 的陣列 `code` 裝 A 到 Z 的次數，初始化為 0。
  - 用 `getline` 讀字串後逐一檢查字元。利用 ASCII 碼的特性，如果是小寫就減掉 `'a'`，大寫就減掉 `'A'`，把它們統整換算成 `0 ~ 25` 的索引值 `a`。
  - 同時準備一個變數 `l`，只要讀到有效的英文字母 `l` 就加 1，這代表「整篇文章的總字母數」（也就是某個字母出現次數的理論最大值）。
* **輸出**：
  - 為了避開寫麻煩的自定義排序 (Custom Sort)，我直接利用雙層迴圈：
  - **外層迴圈**：從最大可能出現的次數 `l`，一路往下遞減扣到 `1`。
  - **內層迴圈**：從 `0` 跑到 `25`（也就是從 A 檢查到 Z）。
  - 只要陣列裡面的次數剛好等於現在外層迴圈的數字，就把它印出來！這樣一來，次數絕對是由大到小，且次數相同時，因為內層迴圈是從 0 跑到 25，自然就會照著 A 到 Z 的順序印出來。

## 3. 注意事項與筆記
- **吃掉換行 (cin.ignore)**：因為一開始是用 `cin >> n` 讀取數字，後面接著要用 `getline` 讀整行字串，所以中間**一定要加上 `cin.ignore()`** 把 Enter 吃掉，不然第一行字串會讀到空的東西。
- **非字母字元的過濾**：題目有很多標點符號跟空白，我在判斷時把初始值設為 `a = -1`，只有真正是字母的時候才會更新 `a` 的值。最後用 `if(a != -1)` 來決定要不要計數。
