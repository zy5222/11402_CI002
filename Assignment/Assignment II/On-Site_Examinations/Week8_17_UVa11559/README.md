# UVa 11559 - Event Planning

### 1. 題目摘要

有一團 $n$ 個人要出去玩，總預算限制是 $b$。現在有 $h$ 家旅館可以選，每家旅館會提供它們的「單人房價」以及接下來 $w$ 個週末的「空床數」。
目標是找出：在床位足夠的週末裡，這團人能夠住進去且「**不超過預算**」的「**最低總花費**」。如果全部都太貴或是都沒床位，就印出 `stay home`。

### 2. 我的解題邏輯

這題我一開始想太多，想把所有的床位資訊都用陣列存起來，但其實**一邊讀取一邊判斷**才是最簡潔的做法。

* **設定計分板**：一開始先設一個 `min_cost = b + 1`，故意讓它比總預算大。最後只要檢查 `min_cost` 有沒有變小（$\le b$），就知道到底有沒有找到合適的旅館。
* **逐家旅館檢查**：
  * 讀到單人房價 `p` 時，馬上算出整團的總花費 `cost = n * p`。
  * 設一個開關 `bool can_stay = false`。
* **丟棄式讀取床位**：用單層迴圈讀取 $w$ 個週末的床位。不需要開陣列存，直接用一個變數 `bed` 接住。只要發現 `bed >= n`，就把開關打開 (`can_stay = true`)。
* **更新最低花費**：只要確認這家旅館有床位 (`can_stay`) 而且花費沒爆表 (`cost <= b`)，就用 `min(min_cost, cost)` 把更便宜的價格記錄下來。

### 3. 筆記與踩坑紀錄

* **Vector 的致命傷 (Segmentation Fault)**：
  我原本想用 `vector<int> v;` 來存床位，但忘記給它大小就直接 `cin >> v[i]`，這樣程式會直接崩潰。後來發現這題根本不需要存床位資訊，讀過去判斷完就可以丟了，省空間又省麻煩。
* **資料必須讀乾淨**：
  在檢查床位時，就算第一個週末就發現床位夠了，內層的 `for` 迴圈還是得乖乖跑完。如果不小心加了 `break` 提早跳出，剩下的床位數字就會被當成下一家旅館的價格或是下一次測資的 $n$，整個輸入流會大亂，這點要特別小心。
