# UVa 108 - Maximum Sum

### 1. 題目摘要

給定一個 $N \times N$ 的二維整數矩陣，請在其中找出一個「**矩形子陣列**」，使得這個子陣列內部所有數字的總和最大。最後輸出該最大總和。

### 2. 我的解題邏輯

這題是經典的**「最大子陣列和 (Maximum Subarray Sum)」**的 2D 版本。如果用暴力法去窮舉矩形的左上角和右下角座標，再加總裡面的數字，時間複雜度會高達 $O(N^4)$ 甚至是 $O(N^6)$。為了解決這個問題，我們可以使用**「降維打擊」**配合一維的 **Kadane's Algorithm**。

1. 我們利用雙層 `for` 迴圈（`top` 和 `bottom`）來固定矩形的「上邊界」與「下邊界」。
2. **壓縮成一維**：在固定的上下邊界內，把每一個直行（column）的數字由上往下加起來。這樣原本的 2D 矩形問題，就被壓縮成一個長度為 $N$ 的 1D 陣列 `col_sum` 了。
3. **Kadane's 演算法**：針對這個 `col_sum` 陣列，我們跑一次一維的最大子序列和演算法（不斷累加數字，如果當前的累積總和變成負數，就歸零從下個數字重新計算）。找出當下這個上下邊界組合的最大值，並隨時更新全域的 `max_sum`。

### 3. 筆記

* **時間複雜度大降級**：
  透過把「直列」加總壓縮成一維陣列的技巧，時間複雜度成功降到 $O(N^3)$。面對 $N=100$ 的測資，運算次數大約落在 $10^6$，可以非常輕鬆且快速地通過。
* **負數的陷阱**：
  矩陣可能全都是負數。所以在實作時，用來記錄最大值的變數 `max_sum` 絕對不能預設為 0，必須設為一個**極小的負數**（例如 `-1e9`），這樣才能正確找出最大的「負數和」，而不會錯誤地輸出 0。
* **空間優化**：
  這個做法不需要額外建立 2D 陣列來算 Prefix Sum（前綴和），只需要一個大小為 $N$ 的 1D `vector` 就能在迴圈裡重複利用，空間複雜度僅為 $O(N)$。
