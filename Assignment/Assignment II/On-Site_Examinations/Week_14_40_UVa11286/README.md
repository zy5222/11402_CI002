# UVa 11286 - Conformity

### 1. 題目摘要

每位新生必須選擇剛好 5 門課程。一個「課程組合」的受歡迎程度，取決於有多少位新生選擇了完全相同的這 5 門課（選課順序不重要）。
請找出最受歡迎的課程組合（可能有多個組合並列第一），並輸出**選擇這些最受歡迎組合的「新生總人數」**。當輸入 $N=0$ 時結束程式。

### 2. 解題邏輯

這題的核心在於：**如何判斷兩個學生選的 5 門課是「相同」的組合？**
因為輸入的順序可能不同（例如 `1 2 3 4 5` 和 `5 4 3 2 1` 其實是同一組），我們必須先對資料進行「正規化」。

1. **排序正規化**：讀取每位學生的 5 門課程後，立刻對這 5 個數字進行由小到大的排序。這樣一來，相同的課程組合就必定會長得一模一樣。
2. **統計頻率 (Map)**：在 C++ 中，`std::vector` 原生就支援比較運算（`==`, `<`, `>`），因此我們可以直接使用 `map<vector<int>, int>`，把排序後的陣列當作 Key，出現次數當作 Value，來統計每個組合的人數。
3. **找出最大值並加總**：
   * 走訪整個 Map，找出最高的出現次數 `max_count`。
   * **陷阱注意**：如果有兩個不同的組合都出現了 3 次（並列最高），那麼答案應該是 $3 + 3 = 6$ 人。
   * 我們可以在走訪 Map 時一併更新：如果遇到更高的次數，就重置 `max_count` 和 `total_students`；如果遇到與 `max_count` 相同的次數，就把該組合的人數累加到 `total_students` 中。

### 3. 筆記

* **時間與空間複雜度**：
  每位學生的 5 門課排序時間為 $O(1)$（常數時間）。將長度為 5 的 `vector` 插入 `map` 的時間複雜度為 $O(\log N)$。對於 $N$ 位新生，整體時間複雜度為 $O(N \log N)$，在 $N \le 10000$ 的限制下執行速度極快。
* **為什麼用 `std::map` 而不是 `std::unordered_map`？**
  雖然 `unordered_map` 理論上是 $O(1)$，但 C++ 標準庫並未提供 `std::vector` 的預設雜湊函式（Hash function）。如果要用 `unordered_map` 必須自己手寫 Hash 結構，會讓程式碼變得冗長。由於這題的 $N$ 不大，直接使用基於紅黑樹的 `std::map` 是最乾淨優雅的選擇。
