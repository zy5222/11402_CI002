# UVa 11389 - The Bus Driver Problem

### 1. 題目摘要
某公司有 $N$ 條早班路線（長度為 $S_i$）與 $N$ 條晚班路線（長度為 $D_i$），需要指派 $N$ 位司機，每位司機各負責一條早班與一條晚班路線。
每位司機的薪資計算方式為：
* 基本薪資 = 該司機所負責的「早班與晚班路線中較長的那一條」之長度，即 $\max(S, D)$。
* 額外費用 = 由於早晚班路線長度不同，司機需要額外往返調度，費用為 $2 \times |S - D|$。

請計算公司付給所有司機的「最小總花費」。

### 2. 我的解題邏輯
這是一題透過數學化簡後轉化為「排序貪婪配對」的題目。

1. **化簡單一司機的花費公式**：
   每位司機的花費為
   $$\text{cost} = \max(S, D) + 2|S - D|$$
   這個式子可以拆解成兩個部分：$\max(S,D)$（基本薪資）與 $2|S-D|$（額外調度費），兩者都隨著 $S, D$ 之間的差距增大而增大。

2. **排序貪婪配對 (Greedy Pairing)**：
   * 將早班路線長度陣列 $S$ 由小到大排序。
   * 將晚班路線長度陣列 $D$ 由小到大排序。
   * 依序將排序後的 $S_i$ 與 $D_i$ 一一配對（第 $i$ 小配第 $i$ 小）。

3. **加總計算答案**：
   對每一組配對 $(S_i, D_i)$ 計算 $\max(S_i, D_i) + 2|S_i - D_i|$，將所有 $N$ 組的花費加總即為該測試案例的最小總花費。

### 3. 筆記
* **為什麼「同序排序配對」是最優解？**：
  無論是 $\sum \max(S_i, D_i)$ 還是 $\sum |S_i - D_i|$，根據排序不等式 (Rearrangement Inequality) 的延伸結論，當兩個陣列都依照「相同方向」排序（同為遞增或同為遞減）後再逐一配對，可以同時讓兩者的總和達到最小值。因為這兩項在同一個配對下同時被最小化，因此「兩陣列同序排序後一一配對」即為全域最佳解，不需要額外的最佳化搜尋。

* **使用 long long 的原因**：
  路線長度與測資規模可能使總花費超過 32 位元整數的範圍，因此採用 `long long` 來避免溢位。

* **時間複雜度**：
  排序耗時 $O(N \log N)$，配對加總耗時 $O(N)$，總時間複雜度為 $O(N \log N)$，可輕鬆應付題目給定的資料規模。
