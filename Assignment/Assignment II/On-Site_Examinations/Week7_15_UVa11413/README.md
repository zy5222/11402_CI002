# UVa 11413 - Fill the Containers

### 1. 題目摘要

桌上有 $n$ 杯牛奶（容量不同），現在要把這些牛奶**照順序**倒進 $m$ 個大桶子裡面。
目標是找出：在所有能夠順利把牛奶裝完的方案中，大桶子的「**最小可能容量**」是多少？

### 2. 我的解題邏輯

這題看起來像是在考陣列分割，但其實暴力法根本算不出來。最佳解法是「**對答案進行二分搜 (Binary Search on Answer)**」——與其慢慢算，不如直接「猜」容量，然後驗證對不對。

* **設定猜測範圍（邊界）**：
  * **下界 (`left`)**：至少要是陣列裡的「最大值」。因為如果桶子比最大那杯牛奶還小，那杯就永遠倒不進去了。
  * **上界 (`right`)**：所有牛奶的「總和」。這是最極端的狀況，也就是只用 1 個超大桶子把全部牛奶裝完。
* **二分搜主體**：
  每次取中間值 `mid = left + (right - left) / 2` 當作這次猜測的容量限制，交給 `can_fill` 函式去模擬。
  * 如果 `can_fill` 回傳 `true`（代表容量 `mid` 夠裝）：先把這組可行的答案 `ans = mid` 記下來，然後貪心一點，把上界往下壓 (`right = mid - 1`)，試試看能不能用更小的桶子。
  * 如果回傳 `false`（代表裝不下）：代表桶子容量給太小了，必須把下界往上拉 (`left = mid + 1`)。
* **貪心模擬 (`can_fill` 函式)**：
  負責驗證老闆猜的容量對不對。從頭開始遍歷牛奶，裝得下就繼續加 (`cur_sum += v[i]`)；如果這杯倒下去會超過 `cap`，就只好拿一個新桶子 (`count++`)，並把這杯裝進新桶子 (`cur_sum = v[i]`)。最後檢查總共用掉的桶子數有沒有 $\le m$ 即可。

### 3. 筆記與踩坑紀錄

* **下界不是 0 也不是 1**：一開始很容易直覺把二分搜的 `left` 設成 0，但這樣在跑 `can_fill` 的時候會死迴圈或出錯。下界一定要是 `max_val`。
* **防溢位的寫法**：算中間值的時候，寫 `mid = left + (right - left) / 2` 可以避免測資太大的時候 `left + right` 直接爆掉超過 `int` 的上限。
* **這是一種固定套路**：以後只要看到題目要求「最大化最小值」或「最小化最大值」（像是這題找最小容量），可以想到是這種「二分搜答案 + 寫一個 check 函式驗證」的題型。
