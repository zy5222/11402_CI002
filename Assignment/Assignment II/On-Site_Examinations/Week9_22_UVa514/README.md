# UVa 514 - Rails

### 1. 題目摘要

有 $N$ 節火車車廂，依序以 $1, 2, \dots, N$ 的編號從 A 方向駛入車站。車站內部是一個具有「**後進先出 (LIFO)**」特性的死胡同（可以視為一個堆疊 Stack）。火車可以選擇直接開往 B 方向，或者先進入車站停靠，之後再從車站開往 B 方向。
給定一個 B 方向的目標車廂排序，請判斷這個排序是否合法（能否透過車站的調度產生）。合法輸出 `Yes`，否則輸出 `No`。

### 2. 我的解題邏輯

這題完美契合**「堆疊 (Stack)」**的資料結構。我們只需要模擬車廂進站與出站的過程即可。

1. 我們建立一個 `stack<int>` 來模擬車站，並設定一個變數 `current_car = 1` 代表目前準備從 A 方向進來的車廂編號。
2. 依序讀取目標陣列中的每一個車廂編號 `target[i]`。
3. **進站模擬**：如果堆疊是空的，或者堆疊最上方的車廂不是我們現在需要的 `target[i]`，我們就持續把 `current_car` 開進車站（Push），並讓 `current_car` 遞增，直到找到需要的車廂，或是所有車廂都進站為止。
4. **出站判定**：
   * 如果此時堆疊最上方剛好是 `target[i]`，代表調度成功，把這節車廂開往 B 方向（Pop）。
   * 如果堆疊最上方不是 `target[i]`（且 A 方向也沒車了），這意味著我們需要的車廂被壓在車站深處，調度失敗，直接判定為不合法。

### 3. 筆記

* **輸入輸出的格式陷阱**：
  這題的輸入格式有兩層迴圈：第一層是測資大小 $N$（當 $N=0$ 結束程式），第二層是各組目標序列（當序列第一個數字是 `0` 時結束該區塊）。並且，**每個 $N$ 的區塊結束後，必須額外輸出一行空白行**，這是初學者在這題最容易吃 Presentation Error (PE) 的地方。
* **時間複雜度**：
  每節車廂最多只會進站（Push）一次、出站（Pop）一次。所以對於長度為 $N$ 的序列，判斷的時間複雜度為 $O(N)$，非常有效率。
