# UVa 10821 - Constructing BST

### 1. 題目摘要

給定節點總數 $N$（節點值為 $1$ 到 $N$）以及二元搜尋樹的最大高度限制 $H$。
請建構出一棵符合條件的二元搜尋樹，並印出其**前序遍歷 (Pre-order Traversal)**。
為了保證答案唯一，題目要求輸出的前序遍歷必須是「**字典序最小**」的排列。如果無法在高度 $H$ 內塞入 $N$ 個節點，則輸出 `Impossible.`。當 $N=0, H=0$ 時結束程式。

### 2. 解題邏輯

這題不用真的把樹建出來，只需利用遞迴和區間推算就能直接印出前序遍歷的答案。

1. **極限判斷 (Impossible)**：
   一棵高度為 $H$ 的二元樹，最多只能容納 $2^H - 1$ 個節點。如果 $N > 2^H - 1$，絕對不可能建得出來。
2. **貪婪選擇 Root (字典序最小)**：
   前序遍歷的順序是 `中 -> 左 -> 右`。要讓字典序最小，我們必須讓當前區間的**根節點 (Root) 越小越好**。
3. **右子樹的容量限制**：
   假設當前可以使用的數字區間是 $[L, R]$，高度限制為 $H$。
   如果我們選 $X$ 當根節點，那麼右子樹的區間就是 $[X+1, R]$，且右子樹的最大高度剩下 $H-1$。
   高度 $H-1$ 的右子樹，**最多只能塞 $2^{H-1} - 1$ 個節點**。
   這代表：右子樹的節點數 $(R - X)$ 絕對不能超過 $2^{H-1} - 1$。
   移項後得到：$X \ge R - (2^{H-1} - 1)$。
   同時，$X$ 當然不能小於左邊界 $L$。
   因此，為了讓 $X$ 越小越好，我們直接取：**$X = \max(L, R - (2^{H-1} - 1))$**。
4. **遞迴列印**：
   算出 $X$ 後直接印出來（滿足前序的中），接著先遞迴左子樹 $[L, X-1]$，再遞迴右子樹 $[X+1, R]$，高度皆為 $H-1$。

### 3. 筆記

* **位元運算的溢位陷阱**：
  計算 $2^H - 1$ 時，通常會用 `(1 << H) - 1`。但題目 $H$ 的值可能高達 30，如果用標準的 32-bit `int` 去算 `1 << 30` 或更高，很容易發生溢位（Overflow）。保險起見，可以改用 `long long` 的 `1LL << H`，或者當 $H \ge 30$ 時直接視為容量無限大（因為 $N$ 最大只有 10000）。
* **免建樹的時間複雜度**：
  直接透過數學推導找出每一個節點的位置，時間複雜度等於印出 $N$ 個節點的時間 $O(N)$，極度快速且不浪費任何空間。
