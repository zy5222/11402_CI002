# UVa 11988 - Broken Keyboard (a.k.a. Beiju Text)

### 1. 題目摘要

你在使用一個鍵盤壞掉的電腦打字。在打字的過程中，鍵盤的 `Home` 鍵（將游標移到最前方）和 `End` 鍵（將游標移到最後方）會不時地自動觸發。
題目會給定一串字串，其中 `[` 代表 `Home` 鍵被按下，`]` 代表 `End` 鍵被按下。請輸出最終顯示在螢幕上的正確文字。
字串長度最大可達 100,000。

### 2. 解題邏輯

這題最大的陷阱在於字串插入的「時間複雜度」。
如果你使用普通的字串 `std::string` 或陣列 `std::vector`，當游標在最前方（`Home`）並不斷插入新字元時，每一次插入都需要把後面的所有字元往後挪動。在最差情況下，時間複雜度會高達 O(N^2)，面對 100,000 長度的測資絕對會 Time Limit Exceeded (TLE)。

**破局關鍵：使用連結串列 (Linked List)**
要在資料的中間或前端進行極速的「插入」操作，最完美的資料結構就是雙向連結串列 `std::list`。

1. 我們建立一個 `list<char>` 來儲存最終的文字。
2. 建立一個游標（Iterator）`cursor`，用來記錄目前打字的位置。
3. 掃描輸入的字串：
   * 遇到 `[`：將 `cursor` 移動到 `list.begin()`（最前方）。
   * 遇到 `]`：將 `cursor` 移動到 `list.end()`（最後方）。
   * 遇到一般字元：將字元插入到 `cursor` 目前所在的位置，`std::list` 的插入操作會自動將游標保持在正確的相對位置。

### 3. 筆記

* **極致的 O(1) 插入**：
  使用 `std::list.insert()`，無論游標在哪裡，插入一個字元的時間複雜度永遠是 O(1)。整個字串掃描一次，總時間複雜度降至完美的 O(N)，可以輕鬆秒殺這題。
* **Iterator (迭代器) 的特性**：
  在 C++ 的 `std::list` 中，`insert(iterator, value)` 會將元素插入到該迭代器指向的位置的「前方」，這剛好完美符合我們打字時游標的運作邏輯（新打的字會出現在游標前面，游標跟著往後推）。
