# UVa 732 - Anagrams by Stack

### 1. 題目摘要

給定一個來源字串 (Source) 與一個目標字串 (Target)，請找出所有可能的堆疊操作序列，能夠將來源字串轉換成目標字串。
* `i` 代表 Push（將字元推入堆疊）。
* `o` 代表 Pop（將字元從堆疊彈出並輸出）。
對於每組測資，必須以 `[` 開始，並以 `]` 結束。輸出的操作序列必須以**字典序**排列（即優先嘗試 `i`，再嘗試 `o`），且操作字元之間需有一個空白。

### 2. 解題邏輯

這題不能只用單純的堆疊模擬，因為可能有多種不同的操作順序都能達到目標，所以必須使用**深度優先搜尋 (DFS) / 回溯法 (Backtracking)** 來窮舉所有可能的分支。

我們在遞迴過程中需要維護兩個指標：`s_idx` (來源字串目前讀到哪) 與 `t_idx` (目標字串目前匹配到哪)，以及模擬的堆疊狀態和當前的操作路徑。

在每一層遞迴中，我們有兩個選擇：
1. **Push (`i`)**：只要來源字串還沒被全部推入堆疊（`s_idx < source.length()`），我們就可以選擇把下一個字元 Push 進去。
2. **Pop (`o`)**：只要堆疊不是空的，**且**堆疊最上方的字元剛好等於目標字串當前需要的字元（`stack_top == target[t_idx]`），我們就可以選擇 Pop 出來匹配。

**終止條件**：當 `t_idx == target.length()` 時，代表所有字元都成功匹配出棧，我們就可以將收集到的操作路徑印出來。

### 3. 筆記

* **字典序的要求**：
  題目要求輸出需依照字典序，因為 `i` 的 ASCII 碼小於 `o`，所以在寫 DFS 時，**務必先寫 Push (`i`) 的遞迴分支，再寫 Pop (`o`) 的遞迴分支**，這樣自然印出來就會是正確的字典序。
* **字串代替 Stack 容器的實作巧思**：
  在 C++ 實作 DFS 時，如果直接傳遞 `std::stack`，在回溯時需要頻繁地 push 和 pop 來復原狀態。一個小巧思是直接用 `std::string` 來模擬堆疊（`string.back()` 就是 `top()`，字串相加就是 `push`），這樣在傳遞參數時直接利用 Pass by Value 產生新拷貝，可以省去手動還原狀態的麻煩，程式碼也會極度乾淨。
* **初步剪枝 (Pruning)**：
  如果來源字串和目標字串長度不同，絕對不可能轉換成功，可以直接跳過 DFS，直接輸出空的 `[` 和 `]` 即可。
