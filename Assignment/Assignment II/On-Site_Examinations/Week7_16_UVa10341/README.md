# UVa 10341 - Solve It

### 1. 題目摘要

這題會給六個整數變數 $p, q, r, s, t, u$，要求我們找出在 $0 \le x \le 1$ 範圍內，能夠讓下面這個方程式等於 0 的解（精準到小數點後 4 位）：
$p * e^{-x} + q * \sin(x) + r * \cos(x) + s * \tan(x) + t * x^2 + u = 0$
如果找不到解，就印出 `No solution`。

### 2. 我的解題邏輯

這個函數在這個區間內是「單調遞減」的（一路往下掉）。只要有這個特性，就可以聯想到套用「**小數版的二分搜尋法**」硬解，不用真的去算數學。

* **把方程式獨立出來**：為了不讓 `main` 迴圈看起來太亂，我直接宣告一個 `double f(double x)` 函式，專門用來代入數字算方程式的結果。
* **先判斷無解 (防呆機制)**：因為函數是一路往下的，如果在起點 $x = 0$ 算出來就已經 $< 0$ 了，或者到了終點 $x = 1$ 算出來還是 $> 0$，那代表曲線絕對不可能跟 $x$ 軸交叉碰到 0，就直接 `continue` 提早結束並印出無解。
* **暴力切半 100 次**：跟整數二分搜不同，小數二分搜不用去煩惱 `+1` 還是 `-1`，也不用設定 `while (left <= right)`。我直接設 `left = 0.0`、`right = 1.0`，然後用 `for` 迴圈固定跑 100 次。每次找中間值 `mid`，如果 `f(mid) > 0` 代表答案還在更右邊 (`left = mid`)，反之則在左半邊 (`right = mid`)。切 100 次之後，區間已經縮小到誤差可以忽略不計了。

### 3. 筆記與踩坑紀錄

* **數學常數跟函式直接 call 內建的**：一開始本來想自己手打一個 `e = 2.7182`，但這樣精確度絕對會出包被判 WA。正確作法是 `#include <cmath>`，然後直接用內建的 `exp(-x)`，包含 `sin`、`cos`、`tan` 也都能直接呼叫。
* **迴圈結束後要印誰**：當迴圈跑完 100 次之後，`left` 和 `right` 就像兩面牆一樣，已經左右夾擊到完全重合了。所以最後直接印出 `left` 就可以了。
* **小數二分搜模板**：這題就是小數二分搜的超經典起手式。之後只要看到「找方程式的解」加上「函數單調遞增/遞減」，就可以聯想到這個解法。
