# UVa 10359 - Tiling

### 1. 題目摘要
有一道高度為 2、寬度為 $N$ 的牆，要用大小為 $1 \times 2$ 的磚塊（可橫放也可直放）完全鋪滿。請計算總共有多少種不同的鋪法，其中 $N$ 最大可達 250，答案會是極大的數字，遠超出一般整數型別範圍。

### 2. 我的解題邏輯
這題與 UVa 900 (Brick Wall Patterns) 是同一種遞迴模型，差別只在於 $N$ 的範圍大很多（最大到 250），因此更需要倚賴大數運算來避免溢位。

1. **遞迴關係推導**：
   考慮寬度為 $N$ 的牆，觀察最右側（第 $N$ 欄）的鋪法情況：
   * **情況一**：用一塊磚直放在最右側那一欄，剩下寬度 $N-1$ 的牆需要鋪滿，貢獻 $f(N-1)$ 種方法。
   * **情況二**：用兩塊磚橫放，分別佔據最右側兩欄的上排與下排，剩下寬度 $N-2$ 的牆需要鋪滿，貢獻 $f(N-2)$ 種方法。
   
   因此可得遞迴式：
   $$f(N) = f(N-1) + f(N-2)$$
   
   邊界條件為 $f(0) = 1$（空牆，視為一種「什麼都不做」的合法鋪法）、$f(1) = 1$（只能直放一塊磚）。這是標準 Fibonacci 數列的形式（只是起始值對應到 $f(0)=1, f(1)=1$）。

2. **大數加法 (Big Integer Addition)**：
   由於 $N$ 最大到 250，$f(250)$ 的位數會非常龐大（約 50 多位數），遠超 `long long` 甚至 `__int128` 的容納範圍，因此必須採用「陣列模擬大數」的方式儲存與相加。
   * 用 `vector<int>` 儲存大數，每個元素代表一個十進位數字，索引 0 為個位數（方便逐位相加與進位）。
   * `add` 函式模擬手動直式加法，逐位相加並處理進位。

3. **預先計算所有答案**：
   由於測試資料可能有多筆查詢，且 $N$ 範圍固定（$0 \le N \le 250$），採用「預先計算 $f(0)$ 到 $f(250)$ 所有結果並存入陣列」的策略，每筆查詢只需 $O(1)$ 直接查表並輸出，避免重複計算造成效能浪費。

### 3. 筆記
* **與 UVa 900 的差異**：
  解題邏輯與遞迴式完全相同，唯一差別在於：
  * 資料範圍從 $N \le 30$ 擴大到 $N \le 250$，使得答案位數大幅增加，大數運算的必要性更加明顯（UVa 900 中即使不用大數，視語言與型別選擇也許還能勉強處理，但這題幾乎必然會溢位）。
  * 邊界條件的定義方式略有不同（這裡採用 $f(0)=1, f(1)=1$ 的標準 Fibonacci 寫法，而非 UVa 900 中 $f(1)=1, f(2)=2$ 的偏移寫法），但兩者本質上代表同一個數列，只是索引方式不同。

* **大數運算的必要性**：
  題目資料規模已經明確要求大數處理，若嘗試用 `long long` 儲存會在 $N$ 約超過 90 左右就發生溢位，因此這題大數運算是「必須」而非「保險起見」的作法。

* **時間複雜度**：
  預先計算 250 個大數加法，每次加法操作的時間複雜度與數字位數成正比（最多約 50~60 位數），整體預處理時間相當輕量；每筆查詢輸出為 $O(1)$ 查表，因此即使測資筆數很多，程式依然能非常快速地完成所有查詢。

* **多筆測試資料 / 結束條件**：
  此題沒有明確的結束標記，直接讀到檔案結尾 (EOF) 為止，因此使用 `while(cin >> N)` 持續讀取直到讀取失敗才結束迴圈。
