# UVa 12347 - Binary Search Tree

### 1. 題目摘要

給定一棵二元搜尋樹 (Binary Search Tree, BST) 的「前序遍歷 (Pre-order Traversal)」結果，請輸出該樹的「後序遍歷 (Post-order Traversal)」。
測資的節點總數不超過 $10000$。

### 2. 解題邏輯

這題的破局關鍵在於熟悉**前序遍歷**與**二元搜尋樹**的特性：
* **前序遍歷**的順序是 `中 -> 左 -> 右`。所以給定的陣列中，第一個數字絕對是整棵樹（或當前子樹）的「根節點 (Root)」。
* **二元搜尋樹**的特性是：左子樹的所有節點值必定小於根節點，右子樹的所有節點值必定大於根節點。

結合這兩個特性，我們可以用**分治法 (Divide and Conquer)** 透過遞迴來解題：
1. 取出當前區間的第一個數字當作 `root`。
2. 往後尋找**第一個大於 `root` 的數字**。這個位置就是「左右子樹的分界點 (Split)」。
3. 分界點左邊的所有數字就是「左子樹」的範圍；分界點右邊的所有數字就是「右子樹」的範圍。
4. 因為題目要求輸出**後序遍歷**（順序為 `左 -> 右 -> 中`），所以我們依序遞迴處理左子樹區間、遞迴處理右子樹區間，最後再印出當前的 `root`。

### 3. 筆記

* **免建樹的空間優化**：
  許多初學者會嘗試宣告 `struct Node` 並真正去 Insert 建立一棵二元樹，然後再來做後序遍歷。這雖然可行，但非常浪費記憶體與執行時間。透過維護陣列的索引 `left` 和 `right`，我們可以「虛擬」地切割子樹，程式碼更短且效能極高。
* **時間複雜度探討**：
  在平均情況下，尋找分界點耗費 $O(\log N)$，整體時間複雜度為 $O(N \log N)$。
  在最差情況下（例如測資已經由小到大排序好，樹退化成一條向右傾斜的直線），每次尋找分界點都需要掃描 $O(N)$ 次，整體時間複雜度為 $O(N^2)$。因為 $N \le 10000$，最差運算量大約為 $10^8$ 的一半，C++ 依然能夠在時限內輕鬆通過。
