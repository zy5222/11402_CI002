# UVa 10405 - Longest Common Subsequence

### 1. 題目摘要

給定兩個字串，請找出這兩個字串的「**最長共同子序列 (Longest Common Subsequence, LCS)**」的長度。
子序列的定義是：從原字串中刪除若干個字元後，剩下字元保持原本的相對順序所形成的新字串。

### 2. 解題邏輯

這是極度經典的動態規劃 (DP) 題型。我們無法使用貪婪法，因為字元匹配的先後順序會影響到整體的長度。

1. **狀態定義**：
   建立一個二維陣列 `dp[i][j]`，代表「第一個字串的前 $i$ 個字元」與「第二個字串的前 $j$ 個字元」的最長共同子序列長度。
2. **初始化**：
   當 $i=0$ 或 $j=0$ 時（代表其中一個是空字串），LCS 的長度必然是 0。所以在宣告 `vector` 時，將大小設為 `(len1 + 1) x (len2 + 1)`，並全預設為 0。
3. **狀態轉移**：
   雙層迴圈遍歷兩個字串的每個字元：
   * **如果字元相同 (`s1[i-1] == s2[j-1]`)**：
     代表我們找到了一個共同字元，這個字元可以接在兩者前綴的 LCS 之後。
     狀態轉移方程：`dp[i][j] = dp[i-1][j-1] + 1`
   * **如果字元不同 (`s1[i-1] != s2[j-1]`)**：
     代表這兩個字元無法同時成為共同子序列的結尾，我們必須捨棄其中一個。因此，我們取「捨棄 `s1` 當前字元」或「捨棄 `s2` 當前字元」兩者中的最大值。
     狀態轉移方程：`dp[i][j] = max(dp[i-1][j], dp[i][j-1])`
4. 最終答案就會落在 `dp[len1][len2]`。

### 3. 筆記

* **輸入讀取的陷阱**：
  這題的字串可能是「空字串」，或者「包含空白字元」。如果使用 `cin >> s1 >> s2`，遇到空白就會被截斷而產生錯誤。因此，**必須使用 `getline(cin, s)` 來整行讀取**。
* **時間與空間複雜度**：
  字串長度最大約為 1000。雙層迴圈的時間複雜度為 $O(N \times M)$，建立二維陣列的空間複雜度也是 $O(N \times M)$。對於 $1000 \times 1000$ 的測資，時間和空間都在合理的範圍內，可以輕鬆 AC。
