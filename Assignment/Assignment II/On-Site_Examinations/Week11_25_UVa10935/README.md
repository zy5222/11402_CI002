# UVa 10935 - Throwing cards away

### 1. 題目摘要

給定 $N$ 張牌，編號 1 到 $N$，由上到下排列成一疊。只要牌堆中至少還有兩張牌，就重複執行以下兩個動作：
1. 把最上面的牌丟掉。
2. 把新出現的最上面的牌移到整疊牌的最底下。
請輸出依序被丟棄的牌的編號，以及最後剩下的那張牌。當輸入為 `0` 時結束程式。

### 2. 解題邏輯

這題的動作邏輯完美契合**「佇列 (Queue)」**的先進先出 (FIFO) 特性。
從最上方拿牌相當於從佇列前端 `pop()`，而把牌放到最底下相當於推入佇列後端 `push()`。

1. 我們建立一個 `queue<int>`，並把 1 到 $N$ 依序推入佇列中。
2. 只要佇列的長度大於或等於 2，就執行迴圈：
   * **丟棄頂部的牌**：印出 `q.front()`，然後 `q.pop()`。
   * **將下一張牌移到底部**：把現在的 `q.front()` 推入底部 `q.push(q.front())`，然後把它從頂部移除 `q.pop()`。
3. 迴圈結束後，佇列中必定只剩下 1 張牌，這張就是最後剩下的牌。

### 3. 筆記

* **格式陷阱 (Edge Case)**：
  這題最容易出錯的地方在於 $N = 1$ 的情況。當只有一張牌時，**不會有任何牌被丟棄**。此時輸出的 `Discarded cards:` 後面不應該有任何數字與逗號，甚至不能有多餘的空白字元。在實作時，利用判斷式控制逗號與空白的輸出，可以完美避開 Presentation Error (PE)。
* **時間複雜度**：
  每次迴圈會移除一張牌，總共需要執行 $N-1$ 次迴圈才能把牌扣到剩下一張，每次 `push` 和 `pop` 的時間複雜度都是 $O(1)$，因此整體時間複雜度為 $O(N)$，執行效率極佳。
