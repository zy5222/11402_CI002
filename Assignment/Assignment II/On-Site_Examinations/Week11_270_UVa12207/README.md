# UVa 12207 - That is Your Queue

### 1. 題目摘要

有一家醫院的看診佇列，總人口數為 $P$（編號從 1 到 $P$ 依序排隊）。醫院會執行 $C$ 個指令，指令分為兩種：
* `N` (Next)：讓排在最前面的人看診。看完診後，這個人會重新排到佇列的「最尾端」。每次執行此指令時，必須輸出看診人的編號。
* `E x` (Emergency)：編號 $x$ 的人發生緊急狀況，直接「插隊」移動到佇列的「最前端」。
給定 $P$ 和 $C$，請模擬這個過程。當 $P=0$ 且 $C=0$ 時結束程式。

### 2. 解題邏輯

這題最大的陷阱在於人口數 $P$ 最大可以高達 $10^9$（十億），如果你乖乖開一個大小為十億的陣列或 Queue 來存 1 到 $P$，絕對會 Memory Limit Exceeded (MLE) 或是 Time Limit Exceeded (TLE)。

**破局關鍵：觀察指令數量 $C$。**
題目規定 $C$ 最大只有 1000。這意味著：即使連續 1000 次都是叫號指令 `N`，我們最多也只會叫到前 1000 個人。排在 1000 名以後的人，在整個程式執行過程中「絕對不可能」被叫到（除非他們發生緊急狀況 `E x`，但發生緊急狀況時我們可以直接把他們拉進佇列）。

1. **縮小初始佇列**：我們一開始只需要把 1 到 $\min(P, C)$ 的人放進佇列即可。
2. **選擇資料結構**：
   因為會有 `E x` 指令，我們需要把原本在佇列中間的 $x$「抽出來」並放到「最前面」。標準的 `std::queue` 無法從中間移除元素，所以這裡最完美的資料結構是**雙向串列 (`std::list`)**。
3. **模擬指令**：
   * 遇到 `N`：取出 `list.front()`，印出來，然後 `pop_front()` 並 `push_back()` 回尾端。
   * 遇到 `E x`：呼叫 `list.remove(x)` 尋找並移除 $x$（如果 $x$ 根本還不在我們初始化的短佇列中，這步就不會有動作），接著再 `list.push_front(x)` 把他塞到最前面。

### 3. 筆記

* **巧用雙向串列**：
  `std::list` 在這題發揮了極大的作用，自帶的 `.remove(x)` 方法可以幫我們在 $O(N)$ 的時間內刪除特定元素，配合 $.push\_front()$ 的 $O(1)$ 插入，完美符合這題的需求。
* **資料型態**：
  雖然 $C \le 1000$，但 $x$ 的編號可以高達 $10^9$，雖然一般 `int`（通常是 32-bit signed）可以裝到 $2 \times 10^9$，但為了保險與好習慣，人口編號的變數建議使用 `long long`。
* **時間複雜度**：
  初始化佇列耗時 $O(C)$。處理指令時，`N` 是 $O(1)$，`E x` 是 $O(C)$（因為需要遍歷長度最多為 $C$ 的 `list` 來刪除），總共 $C$ 個指令，整體時間複雜度為 $O(C^2)$。因為 $C \le 1000$，運算量在 $10^6$ 級別，能夠極速 AC。
