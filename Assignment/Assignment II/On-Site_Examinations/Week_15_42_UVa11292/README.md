# UVa 11292 - Dragon of Loowater

### 1. 題目摘要

有一個惡龍有 $N$ 個頭，每個頭都有一個大小值。你有 $M$ 個騎士，每個騎士有他的身高（騎士只能砍掉身高大於或等於頭部大小的龍頭，且一個騎士只能砍一個頭）。請計算殺掉這條龍的「最小花費」（所有出戰騎士的身高總和），如果無法殺死所有的頭，則輸出 `Loowater is doomed!`。

### 2. 我的解題邏輯

這是一題標準的貪婪策略題：我們應該讓「越小的騎士」去砍「越小的龍頭」，以節省預算。

1. **排序 (Sorting)**：
   將龍頭的大小與騎士的身高分別由小到大排序。
2. **貪婪配對 (Greedy Matching)**：
   使用雙指標 (`d_idx` 指向龍頭，`k_idx` 指向騎士) 從頭開始掃描：
   * 如果當前騎士的身高足以砍下當前龍頭 (`knights[k_idx] >= dragons[d_idx]`)：
     這個組合最划算，我們讓這個騎士出戰，將其身高加到總成本，並同時移到下一個龍頭和下一個騎士。
   * 如果當前騎士身高不足 (`knights[k_idx] < dragons[d_idx]`)：
     這個騎士太矮了，殺不了這個龍頭（也殺不了任何比它大的龍頭），所以此騎士直接淘汰，跳到下一個更高的騎士看看。
3. **終止判定**：
   * 如果所有龍頭都被砍掉了 (`d_idx == n`)，代表任務成功，輸出總成本。
   * 如果龍頭還沒砍完，騎士就先用完了 (`k_idx == m`)，代表無法完成任務，輸出 `Loowater is doomed!`。

### 3. 筆記

* **為什麼貪婪會成功？**：
  這是因為我們每次都選擇「剛好能滿足需求的最小騎士」。如果放棄當前騎士，選用一個更強壯的騎士來砍小頭，等於是浪費了戰力，這對砍後面的大頭極為不利。
* **時間複雜度**：
  排序耗時 $O(N \log N + M \log M)$，掃描過程耗時 $O(N + M)$，總時間複雜度為 $O(N \log N + M \log M)$。這對於 $N, M \le 20000$ 的限制綽綽有餘。
