# UVa 11349 - Symmetric Matrix

### 1. 題目摘要

給定一個 $N \times N$ 的矩陣，請判斷它是否為「對稱矩陣」。
但在這題中，對稱的定義有兩個嚴格的條件：
1. **中心對稱**：矩陣中座標 $(i, j)$ 的元素，必須等於座標 $(N-1-i, N-1-j)$ 的元素。
2. **非負數**：矩陣裡面**不可以包含任何負數**。只要有一個負數存在，它就絕對不是對稱矩陣。

輸入的第一行包含測資數量 $T$。每組測資會以 `N = x` 的格式給定矩陣大小，接著給定矩陣元素。
輸出格式為 `Test #t: Symmetric.` 或 `Test #t: Non-symmetric.`。

### 2. 解題邏輯

這題如果不小心用二維陣列處理，在計算對稱座標時很容易搞混或寫錯。最聰明的做法是利用**「降維打擊」**，將二維矩陣當作一維陣列來讀取。

1. **讀取輸入的巧思**：
   因為輸入格式包含字元，例如 `N = 3`，我們宣告兩個 `char` 變數去「吃掉」字元 `N` 和 `=`，直接把後面的數字讀進變數 $N$ 中。
2. **一維陣列轉換**：
   由於矩陣是中心對稱，當我們把它全部攤平成一個長度為 $N \times N$ 的一維陣列時，條件剛好會完美轉換成：`array[k] == array[size - 1 - k]`。這等同於判斷一個字串是否為迴文（Palindrome），邏輯瞬間變得非常簡單。
3. **驗證過程**：
   * 在讀取資料的 `for` 迴圈中，同時檢查有沒有數字小於 0。如果有，立刻標記為不合法（但**必須把剩下的數字讀完**，否則會影響下一筆測資的讀取）。
   * 如果所有數字都是非負數，再跑一半的迴圈（$0$ 到 $size / 2$），檢查陣列頭尾對應的數字是否相同。

### 3. 筆記

* **輸入格式的陷阱**：
  遇到這種混雜字串和數字的輸入（`N = 3`），用 `scanf("N = %d", &n)` 或是 C++ 的 `cin >> dummy >> dummy >> n` 是最穩妥的做法。
* **資料型態防禦**：
  雖然題目沒有明說數字範圍會大到哪裡，但在 UVa 的測資中，有些數字可能會超出 32-bit `int` 的極限。因此，用來裝矩陣元素的變數建議直接開 `long long` 來保平安。
* **時間與空間複雜度**：
  只需要遍歷整個矩陣一次讀取、半次檢查，時間複雜度為 $O(N^2)$。使用一維陣列儲存，空間複雜度同樣為 $O(N^2)$。
