# UVa 11450 - Wedding Shopping

### 1. 題目摘要

你有一個總預算 $M$，並且需要購買 $C$ 種不同類別的衣服（每種「必須」且「只能」買一件）。
每種衣服有許多不同的款式，每種款式都有各自的價格。
請計算出在不超過總預算 $M$ 的情況下，你**最多能花多少錢**。如果無論怎麼組合都會超出預算，則輸出 `no solution`。

### 2. 解題邏輯

這題無法使用「貪婪演算法 (Greedy)」（例如每次都挑最貴且買得起的），因為局部最貴不代表全域最佳解，有可能會導致後面買不起其他衣服，或是留下太多零錢。這是標準的**動態規劃 (DP)** 題型。

1. **狀態定義**：
   建立一個二維布林陣列 `dp[g][b]`，代表「在考慮前 $g$ 種衣服時，是否有可能剛好花費 $b$ 元」。
2. **初始化**：
   一開始什麼都沒買，所以 `dp[0][0] = true`（花費 0 元是可行的），其他所有狀態皆為 `false`。
3. **狀態轉移**：
   當我們考慮第 $g$ 種衣服的某個價格 $p$ 時，我們去檢查「上一層（第 $g-1$ 種衣服）」的所有可行花費 $b$。
   如果 `dp[g-1][b]` 是 `true`（代表之前可以剛好花 $b$ 元），且加上這件衣服的價格沒有超支（$b + p \le M$），那麼 `dp[g][b + p]` 就會變成 `true`。
4. **尋找答案**：
   處理完所有 $C$ 種衣服後，我們只要去檢查最後一層 `dp[C]`，從最大預算 $M$ 一路往下檢查到 0。第一個遇到 `true` 的金額，就是我們的最大花費。如果全部都是 `false`，代表無解。

### 3. 筆記

* **時間複雜度極佳**：
   最多有 20 種衣服、每種最多 20 個款式，預算最大 200。我們利用 DP 記錄狀態，時間複雜度為 $O(C \times K \times M)$，最大運算次數僅約為 $20 \times 20 \times 200 = 80000$ 次，執行速度極快。
* **空間複雜度的優化（降維）**：
   在程式碼中我使用了二維陣列 `dp[25][205]`，因為數字很小所以空間非常夠用。如果預算 $M$ 很大，這題其實可以壓縮成一維陣列（只保留「上一層」與「當前層」），將空間複雜度降至 $O(M)$。
