# UVa 1121 - Subsequence

### 1. 題目摘要

給定一個長度為 $N$ 的正整數數列，以及一個目標值 $S$。
請在數列中找出一段「**連續的子序列**」，使得這段子序列的總和 $\ge S$，並且要求這個子序列的「**長度越短越好**」。如果找不到任何符合條件的子序列，則輸出 0。

### 2. 我的解題邏輯

這題是標準的**「滑動視窗 (Sliding Window) / 雙指標」**題型。如果用雙層 `for` 迴圈暴力尋找（從每個起點往後加），時間複雜度會高達 $O(N^2)$，測資太大時一定會 Time Limit Exceeded (TLE)。

1. 我們維持一個視窗，由 `left` 和 `right` 兩個指標控制。
2. **右邊界擴張**：`right` 指標不斷往右移動，把數字加進 `current_sum` 裡面。
3. **左邊界收縮**：當 `current_sum >= s` 時，代表我們找到一組可行的連續子序列了！這時候我們把目前的長度 `right - left + 1` 記錄下來（保留最短的），然後嘗試把 `left` 指標往右移（扣掉最左邊的數字），看看「縮短後的視窗」是不是依然 $\ge S$。

### 3. 筆記

* **時間複雜度大降級**：
  使用滑動視窗技巧，`left` 和 `right` 兩個指標最多都只會把整個陣列走過一次，所以時間複雜度直接降到漂亮的 $O(N)$，幾萬筆測資也能瞬間跑完。
* **預設值的巧思**：
  因為要找的是「最小值」，所以一開始設定 `min_len` 的時候，不能設為 0。我把它設成 `n + 1`（一個絕對不可能達到的長度），這樣最後只要檢查 `min_len` 是不是還是 `n + 1`，就能判斷到底有沒有找到答案（沒找到就印 0）。
* **變數型態要注意**：題目有說 $S$ 可能會大到 $10^8$，陣列全部加起來可能會超過 `int` 的極限，所以 `s` 和 `current_sum` 記得要開 `long long`。
