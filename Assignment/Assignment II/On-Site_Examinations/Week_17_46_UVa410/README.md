# UVa 410 - Station Balance

### 1. 題目摘要
有 $N$ 個裝有石墨的容器，每個容器都有一個重量。需要把這 $N$ 個容器分裝到火車的兩個車廂（兩側），其中一側放 $\lfloor N/2 \rfloor$ 個容器，另一側放剩下的 $N - \lfloor N/2 \rfloor$ 個容器。
目標是讓「兩側總重量的差距越小越好」，並依序輸出兩組容器的編號、重量，以及各組的平均重量。

### 2. 我的解題邏輯
這題本質是「固定組別大小的子集和最佳化」問題：在大小固定為 $K = \lfloor N/2 \rfloor$ 的子集中，找出總和最接近全體總和一半的組合。

1. **狀態定義 (DP)**：
   令 `dp[i][j][s]` 表示「考慮前 $i$ 個容器，選出 $j$ 個容器，其重量和是否能恰好等於 $s$」（布林值）。
   * 轉移時，第 $i$ 個容器可以選或不選：
     * 不選：`dp[i][j][s] = dp[i-1][j][s]`
     * 選：`dp[i][j][s] |= dp[i-1][j-1][s-w[i-1]]`（前提是 $s \ge w[i-1]$）

2. **尋找最佳總和**：
   全體總重為 `total`。掃描所有可行的 `s`（即 `dp[n][K][s] == true`），找出讓 `|total - 2s|` 最小的那個 `s`，這代表 A 組總和為 `s`、B 組總和為 `total - s` 時，兩組差距最小。

3. **回溯重建分組 (Reconstruction)**：
   從 `dp[n][K][bestS]` 開始，由後往前判斷第 $i$ 個容器是否「必須被選」：
   * 若「不選第 $i$ 個容器」這條路徑（`dp[i-1][j][s]`）不可行，但「選」的路徑可行，就代表這個容器一定要分到 A 組。
   * 依此規則逐步回推出 A 組的所有容器，其餘容器自動歸為 B 組。

4. **排序與輸出**：
   兩組容器各自依照「原始編號」由小到大排序後輸出，並計算各組平均重量（取至小數點後兩位）。

### 3. 筆記
* **為什麼用三維 DP 而非單純背包**：
  一般子集和問題只需要二維 `dp[j][s]`（選了幾個、和是多少），但因為需要在「回溯重建解」時準確判斷每個容器的取捨，保留 `dp[i][...]` 這一維（代表考慮到第幾個容器）可以避免滾動陣列覆蓋掉重建所需的歷史資訊，寫法更直觀、不容易出錯。

* **複雜度分析**：
  狀態數為 $O(N \times K \times \text{total})$，其中 $N \le 20$、$K \le 10$、`total` 最多約 $20 \times 100 = 2000$，因此整體時間與空間複雜度約為 $O(N^2 \times \text{total})$，在此資料規模下完全可以接受。

* **輸出格式細節**：
  * 每個測試案例以 `Set #X` 開頭，案例編號從 1 開始。
  * 兩組之間需要輸出一個空行分隔，不同測試案例之間也需要空行分隔（但第一個案例前不可有多餘空行）。
  * 平均重量需固定輸出小數點後兩位（`%.2f`），即使整除也要顯示 `.00`。

* **輸入結束條件**：
  當讀到 $N = 0$ 時代表輸入結束，程式應立即停止讀取與輸出。
