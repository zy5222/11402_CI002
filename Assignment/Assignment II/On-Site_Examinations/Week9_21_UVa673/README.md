# UVa 673 - Parentheses Balance

### 1. 題目摘要

給定包含小括號 `()` 與中括號 `[]` 的字串，請判斷這些括號是否正確配對且合法嵌套。
* 如果括號完全合法配對，輸出 `Yes`。
* 如果不合法，輸出 `No`。
* **空字串也視為合法的配對**（這是一個容易忽略的隱藏條件）。

### 2. 解題邏輯

這題是標準的**「堆疊 (Stack)」**應用題。括號的特性是「後進先出 (LIFO)」，最後出現的左括號，必須最先被對應的右括號閉合，這與 Stack 的行為完美契合。

1. 我們建立一個字元堆疊（Stack）。
2. **遇到左括號 `(` 或 `[`**：一律把它推入堆疊中（Push）。
3. **遇到右括號 `)` 或 `]`**：
   * 檢查堆疊是否為空。如果是空，代表沒有對應的左括號，直接判定不合法。
   * 檢查堆疊最上層（Top）的字元是否與當前的右括號配對。如果配對成功，就把最上層的左括號彈出（Pop）；如果不配對，判定不合法。
4. **字串掃描結束**：檢查堆疊是否為空。如果清空了代表全數配對成功；如果還有剩餘的左括號，代表不合法。

### 3. 筆記

* **輸入讀取的陷阱**：
  因為題目明確包含「空字串」，如果你用 `cin >> s` 讀取，它會直接跳過空白字元和空行，導致你讀到下一筆測資，發生嚴重的錯誤（Wrong Answer）。必須使用 `getline(cin, s)` 來整行讀取，包含空行。
* **處理換行字元**：
  讀取最一開始的測資數量 $N$ 時，`cin >> n` 會在輸入緩衝區留下一個換行字元 `\n`。在開始跑 `getline` 迴圈之前，必須先呼叫 `cin.ignore()` 把這個換行字元吃掉，否則第一筆測資會直接讀到一個空字串。
* **時間與空間複雜度**：
  只需要掃描字串一次，時間複雜度為 $O(L)$（$L$ 為字串長度）。堆疊最多儲存 $L$ 個字元，空間複雜度也是 $O(L)$。
