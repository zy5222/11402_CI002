# UVa 10783 - Odd Sum

## 1. 題目摘要
簡單來說，就是給定一個範圍 `[a, b]`，把這範圍裡面的「奇數」全部加總起來。
一開始會先輸入一個數字代表有幾組測試資料，接著每組就給 `a` 跟 `b`。

## 2. 解題邏輯
* **處理測資**：先弄個外層迴圈跑總筆數。這個迴圈變數 `i` 剛好可以用來印最後題目要求的 `Case i:`。
* **找奇數並加總**：
  * 在內層寫一個迴圈，讓數字從 `a` 一路跑到 `b`。
  * 判斷奇數的方式就是 `j % 2 == 1`。
  * 如果是奇數，就加進 `total` 裡面；如果是偶數，就直接 `continue` 跳過。（雖然不寫 `continue` 程式碼也能跑，但寫上去比較符合我當下「看到偶數就略過」的直覺）。

## 3. 注意事項與筆記
- **不用檢查大小**：題目保證了 `a <= b`，所以不用像之前寫 UVa 100 那樣還要自己用 `swap` 交換範圍。
- **資料型態**：範圍很小（數字最大才 100），全部加起來頂多兩三千，所以用一般的 `int` 裝就可以了，不用怕Overflow。
- **輸出格式**：記得 `Case X:` 後面有一個空白，印的時候別漏了。
