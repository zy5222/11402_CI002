# UVa 10446 - The Marriage Interview

### 1. 題目摘要
給定一行字串，請找出該字串「最短的前綴 (prefix)」長度，使得這個前綴恰好包含整個字串中所有「不同的字元種類」各至少一次。輸出此最短前綴的長度。

### 2. 我的解題邏輯
這題的核心是「一次線性掃描」搭配「集合 (set) 計數不同字元種類」的技巧。

1. **統計整個字串需要包含的字元種類數**：
   先用一個 `set<char>` 掃過整個字串，得到所有出現過的不同字元，其數量記為 `need`，這代表「前綴必須蒐集到的字元種類總數」。

2. **由左至右掃描，動態累積已出現的字元種類**：
   再用另一個 `set<char> seen` 從字串開頭開始逐字元掃描：
   * 每讀入一個字元就將其加入 `seen` 集合（`set` 會自動去除重複字元）。
   * 每加入一個字元後，檢查 `seen.size()` 是否已經等於 `need`。
   * 一旦相等，代表「目前累積到這個位置為止的前綴」已經包含了所有字元種類，此時的位置（索引 + 1，因為長度從 1 開始算）即為答案，立刻跳出迴圈。

3. **輸出結果**：
   輸出找到的最短前綴長度 `ans`。

### 3. 筆記
* **為什麼從左掃描一定能找到最短前綴**：
  因為「前綴」的定義本身就是從字串開頭開始連續的一段，所以隨著掃描位置往右移動，`seen` 集合只會單調增加（不會減少），因此第一次讓 `seen.size() == need` 成立的位置，必然就是「最短」滿足條件的前綴終點，不需要額外比較其他位置。

* **使用 `set` 而非陣列計數的考量**：
  由於字元種類有限（通常為英文字母或可見字元），其實也可以用一個布林陣列 (`bool seen[256]`) 搭配計數器來取代 `set`，效率會更高（$O(1)$ 插入與查詢，而非 `set` 的 $O(\log N)$）。但考量到題目資料量通常不大，使用 `set` 寫法更直觀清楚，效能差異可忽略不計。

* **時間複雜度**：
  統計整體不同字元種類耗時 $O(L \log L)$（$L$ 為字串長度），掃描累積過程每次插入 `set` 操作耗時 $O(\log L)$，最壞情況下需掃描整個字串，因此單筆測資總時間複雜度約為 $O(L \log L)$，對一般字串長度而言相當快速。

* **讀取整行字串的注意事項**：
  由於字串中可能包含空白字元（取決於題目實際測資），因此使用 `getline(cin, s)` 讀取整行，而非 `cin >> s`（後者會在空白處截斷），並加入「跳過空行」的判斷以避免處理到無效的空字串。

* **多筆測試資料 / 結束條件**：
  此題沒有明確的結束標記，直接讀到檔案結尾 (EOF) 為止，因此使用 `while(getline(cin, s))` 持續讀取直到讀取失敗（EOF）才結束迴圈。
