# UVa 10041 - Vito's Family

### 1. 題目摘要

黑手黨老大 Vito 想搬到紐約，他有 $R$ 個親戚住在同一條街上。給定這 $R$ 個親戚的門牌號碼（一維座標），Vito 想找到一個「最佳位置」來建立自己的家，使得他的家到所有親戚家的**距離總和最小**。
請輸出這個最小的距離總和。

### 2. 解題邏輯

這題是一個標準的數學問題。初學者常常會誤以為要找「平均數 (Mean)」，但實際上，平均數是用來讓「距離的平方和」最小化。
要讓**「絕對距離的總和」**最小化，最佳的選擇是**「中位數 (Median)」**。

1. 將讀取進來的所有親戚門牌號碼存入陣列。
2. 將陣列由小到大進行排序。
3. 找出中位數：在陣列索引中，直接取 `R / 2` 的位置即為中位數。
4. 跑一個迴圈，把陣列中每個元素與中位數相減取絕對值 (`abs`) 並加總。
5. 輸出的總和即為答案。

### 3. 筆記

* **中位數的奇偶數問題**：
  * 當數量是奇數時，中位數只有一個，這毫無懸念。
  * 當數量是偶數時（例如有 4 個人落在座標 1, 2, 7, 8），中位數雖然理論上落在第 2 和第 3 個元素之間，但實際上，**只要 Vito 的家建在第 2 到第 3 個元素之間的「任何一點」（包含端點），總距離都會是一樣的最小值**。因此，我們在寫程式時，不需要特別區分奇偶數，直接取 `streets[R / 2]` 當作基準點去算距離，答案保證正確。
* **時間複雜度**：
  尋找中位數最快的方法是快速選擇演算法 $O(N)$，但因為這題測資數量 $R$ 最大只有 500，我們直接使用內建的 `std::sort` 排序 $O(R \log R)$ 加上一次 $O(R)$ 的迴圈加總，程式碼不僅極短，而且效能也完全足以應付時限，是 CP 值最高的寫法。
