# LeetCode 860 - Lemonade Change

## 1. 題目摘要

在檸檬汁攤位中，每杯檸檬汁售價固定為 **5 元**。

顧客依序付款，每位顧客只會使用以下三種鈔票：

- `5`
- `10`
- `20`

給定陣列 `bills`，表示每位顧客付款的金額。

請判斷是否能依序找給每位顧客正確的零錢。

若所有顧客皆能成功找零，回傳 `true`；否則回傳 `false`。

例如：

```text
Input:
bills = [5,5,5,10,20]

Output:
true
```

```text
Input:
bills = [5,5,10,10,20]

Output:
false
```

---

## 2. 解題邏輯

本題使用 **Greedy（貪心法）**。

由於每杯檸檬汁價格固定為 **5 元**，因此只需記錄目前手上有多少張：

- `5 元`
- `10 元`

不需要記錄 `20 元`，因為它不會被用來找零。

---

### Step 1：遇到 5 元

若顧客支付：

```text
5 元
```

不需找零，直接增加一張 5 元：

```cpp
five++;
```

---

### Step 2：遇到 10 元

需找：

```text
5 元
```

因此：

若沒有 5 元：

```cpp
five == 0
```

代表無法找零，直接回傳：

```cpp
false;
```

否則：

```cpp
five--;
ten++;
```

---

### Step 3：遇到 20 元

需找：

```text
15 元
```

有兩種方式：

**優先使用：**

```text
10 + 5
```

因為保留更多的 5 元，可增加後續找零成功的機率。

若無法：

再使用：

```text
5 + 5 + 5
```

若兩種方式都無法：

回傳：

```cpp
false;
```

---

### Step 4：全部完成

若所有顧客皆成功找零：

回傳：

```cpp
true;
```

---

## 3. 程式碼實作

```cpp
class Solution {
public:
    bool lemonadeChange(vector<int>& bills) {
        int five = 0;
        int ten = 0;

        for (int bill : bills) {

            if (bill == 5) {
                five++;
            }

            else if (bill == 10) {
                if (five == 0)
                    return false;

                five--;
                ten++;
            }

            else { // bill == 20

                if (ten > 0 && five > 0) {
                    ten--;
                    five--;
                }
                else if (five >= 3) {
                    five -= 3;
                }
                else {
                    return false;
                }
            }
        }

        return true;
    }
};
```

---

## 4. 複雜度分析

| 項目 | 複雜度 |
|------|--------|
| 時間複雜度 | **O(n)** |
| 空間複雜度 | **O(1)** |

其中：

- `n` 為顧客數量。

每位顧客只會處理一次，因此時間複雜度為 **O(n)**。

只使用兩個變數紀錄 5 元與 10 元鈔票數量，因此空間複雜度為 **O(1)**。

---

## 5. 重點整理

- 使用 **Greedy（貪心法）** 模擬找零過程。
- 只需記錄目前持有的 **5 元** 與 **10 元** 鈔票數量。
- 當收到 **20 元** 時，優先找：
  ```text
  10 + 5
  ```
  若不可行，再找：
  ```text
  5 + 5 + 5
  ```
- 優先使用 `10 + 5` 可保留更多 5 元，提高後續找零成功率。
- 每位顧客只需處理一次，因此時間複雜度為 **O(n)**，是本題最佳解法。
