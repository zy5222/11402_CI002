# LeetCode 455 - Assign Cookies

## 1. 題目摘要

假設有：

- 陣列 `g`：每位小孩的**胃口值（Greed Factor）**。
- 陣列 `s`：每塊餅乾的**尺寸（Cookie Size）**。

若餅乾尺寸：

```text
s[j] >= g[i]
```

則該餅乾可以滿足該小孩。

每位小孩最多只能分配一塊餅乾，每塊餅乾也只能分配給一位小孩。

請回傳最多可以滿足多少位小孩。

例如：

```text
Input:
g = [1,2,3]
s = [1,1]

Output:
1
```

Explanation：

只有一位胃口值為 `1` 的小孩能被滿足。

```text
Input:
g = [1,2]
s = [1,2,3]

Output:
2
```

---

## 2. 解題邏輯

本題使用 **Greedy（貪心法）**。

核心想法是：

> **將最小的餅乾優先分配給胃口最小的小孩**，保留較大的餅乾給胃口較大的小孩，以提高整體滿足人數。

---

### Step 1：排序

先將：

- 小孩胃口 `g`
- 餅乾尺寸 `s`

皆由小到大排序：

```cpp
sort(g.begin(), g.end());
sort(s.begin(), s.end());
```

---

### Step 2：使用雙指標

建立兩個指標：

```cpp
i
```

代表目前小孩。

```cpp
j
```

代表目前餅乾。

---

### Step 3：依序分配餅乾

若：

```cpp
s[j] >= g[i]
```

表示目前餅乾可以滿足這位小孩。

因此：

```cpp
count++;
i++;
j++;
```

若：

```cpp
s[j] < g[i]
```

表示餅乾太小。

只能嘗試下一塊較大的餅乾：

```cpp
j++;
```

---

### Step 4：回傳答案

當其中一方已遍歷完成時停止。

最後：

```cpp
count
```

即為最多可滿足的小孩數量。

---

## 3. 程式碼實作

```cpp
class Solution {
public:
    int findContentChildren(vector<int>& g, vector<int>& s) {
        sort(g.begin(), g.end());
        sort(s.begin(), s.end());

        int i = 0;
        int j = 0;
        int count = 0;

        while (i < g.size() && j < s.size()) {
            if (s[j] >= g[i]) {
                count++;
                i++;
                j++;
            } else {
                j++;
            }
        }

        return count;
    }
};
```

---

## 4. 複雜度分析

| 項目 | 複雜度 |
|------|--------|
| 時間複雜度 | **O(n log n + m log m)** |
| 空間複雜度 | **O(1)**（不計排序所需空間） |

其中：

- `n` 為小孩數量。
- `m` 為餅乾數量。

排序分別需 **O(n log n)** 與 **O(m log m)**，之後利用雙指標掃描一次，因此為 **O(n + m)**。整體時間複雜度由排序主導。

---

## 5. 重點整理

- 使用 **Greedy（貪心法）** 搭配排序解題。
- 將小孩胃口與餅乾尺寸皆由小到大排序。
- 使用雙指標依序配對：
  - 若餅乾尺寸足夠，分配給目前胃口最小的小孩。
  - 若餅乾太小，直接嘗試下一塊較大的餅乾。
- 每位小孩與每塊餅乾最多只會處理一次，因此排序後的掃描時間為 **O(n + m)**。
- 此方法是本題最經典且效率最佳的解法。
