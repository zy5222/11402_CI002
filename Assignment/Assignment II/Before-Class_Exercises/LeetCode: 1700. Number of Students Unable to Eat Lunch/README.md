# LeetCode 1700 - Number of Students Unable to Eat Lunch

## 1. 題目摘要
給定兩個長度相同之整數陣列 `students` 與 `sandwiches`，分別代表學生對三明治之喜好與目前堆疊中三明治之種類（$0$ 代表圓形，$1$ 代表方形）。
學生排列為一佇列（Queue），三明治排列為一堆疊（Stack）。
若佇列前端之學生喜愛堆疊頂端之三明治，則取走該三明治並離開佇列；若不喜愛，則放棄該三明治並重新排至佇列最尾端。
此過程將持續進行，直至佇列中所有學生皆無法接受堆疊頂端之三明治為止。請計算並回傳最終無法吃到三明治之學生人數。

---

## 2. 解題邏輯
本題雖涉及佇列與堆疊之操作，但若直接模擬整個流程，最差情況下時間複雜度可能退化。最佳解法為**計算法（Counting）**，利用「學生之順序無關緊要」之特性將時間複雜度最佳化至 $O(n)$。

1. **核心概念分析**：
   由於不喜歡目前三明治的學生會重新排至隊伍尾端，這意味著只要佇列中**還有任何一位**學生想要堆疊頂端的三明治，該三明治最終必將被取走（因為排斥的學生會不斷讓位）。反之，流程會卡死（Halt）的唯一條件是：堆疊頂端的三明治種類，在剩餘的學生中已**無人需要**。

2. **統計需求量**：
   宣告一個長度為 $2$ 之陣列（或兩個獨立變數），統計學生中喜愛圓形（$0$）與方形（$1$）之總人數。

3. **依序匹配堆疊**：
   由左至右遍歷 `sandwiches` 陣列（此動作等同於模擬堆疊頂端之消耗）：
   * 若當前三明治種類之對應學生需求數大於 $0$，則將該需求數減 $1$（代表必定有一名學生最終會將其取走）。
   * 若當前三明治種類之對應學生需求數等於 $0$，代表佇列中所有剩餘學生皆排斥此三明治，導致無人能將其取走，流程強制終止。

4. **回傳結果**：
   迴圈終止時，兩種需求人數之總和（即尚未被滿足之學生總數）即為所求。

---

## 3. 程式碼實作

```cpp
class Solution {
public:
    int countStudents(vector<int>& students, vector<int>& sandwiches) {
        int count[2] = {0, 0};
        
        // 統計學生對各種類三明治之需求人數
        for (int student : students) {
            count[student]++;
        }
        
        // 依序檢驗堆疊中之三明治是否能被消耗
        for (int sandwich : sandwiches) {
            if (count[sandwich] > 0) {
                count[sandwich]--;
            } else {
                // 若無人需要目前堆疊頂端之三明治，流程卡死，直接中斷
                break;
            }
        }
        
        // 剩餘未被滿足之學生人數總和
        return count[0] + count[1];
    }
};
